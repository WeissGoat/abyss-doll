# AI 生成与筛选规范

> **定位：** 规定 AI 图片生成、批次管理、预处理、筛选和 Manifest 状态回填。
> **更新时间：** 2026-05-05

---

## 1. 目标

AI 素材工作流不是一次生成最终资产，而是快速获得可筛选方向稿，再把少量稳定结果整理成工程可引用资源。

```text
prompted -> generated -> selected -> approved -> registered -> validated
```

目录只分两层：`_IncomingAI` 保存生成、预处理和筛选过程，`Approved` 保存正式接入素材。

---

## 2. 提示词生成

Step 2 对 Manifest 中 `Status=todo` 的 entry 填写：

* `PromptEN`
* `NegativePromptEN`
* `Spec`

同时填写 `PromptCN` 作为中文审阅描述。

`PromptEN` 必须使用英文，并且只写视觉语言。中文字段 `SourceFactsCN` 只供内部理解配置事实，不进入绘图提示词。

推荐结构：

```text
asset type + subject + shape/silhouette + material/color + camera/composition + background + style
```

示例：

```text
dark sci-fi, post-apocalyptic industrial fantasy, game item icon, a long tactical blade, narrow sharp blade, black handle, worn industrial metal, subtle cold-blue energy grooves, vertical silhouette, centered single object, transparent background, no text
```

当前风格基准详见 [04_美术风格基准.md](04_美术风格基准.md)。

禁止在 `PromptEN` 和 `NegativePromptEN` 中出现：

* 项目名：如 `魔偶深渊`。
* 玩法黑话：如 `搜打撤`、`塔科夫like`。
* 引擎和 UI 框架：如 `Unity`、`UGUI`。
* 抽象功能诉求：如“服务于界面”“不影响按钮可读性”。
* 游戏策划逻辑：如“站位空间”“战斗敌人卡片”。

如果需要表达这些诉求，必须翻译成视觉语言：

| 错误表达 | 正确视觉表达 |
|---|---|
| 留出 UI 空间 | `large negative space`, `low visual noise`, `simple background` |
| 左右站位空间 | `side-scrolling perspective`, `empty ground across the foreground` |
| 不抢角色主体 | `no main character`, `no large foreground creature`, `soft background contrast` |
| 小尺寸可读 | `bold silhouette`, `high contrast`, `simple shape` |

## 3. `_IncomingAI` 资产工作区

`_IncomingAI` 不再按批次建目录，而是按 `VisualID` 建资产工作区：

```text
UnityClient/Assets/Art/_IncomingAI/<VisualID>/
  raw/
    YYYYMMDD_001.png
    YYYYMMDD_002.png
  processed/
  selected/
  contact_sheet/
  manifest_snapshot.json
  generation.json
  notes.md
```

示例：

```text
UnityClient/Assets/Art/_IncomingAI/item_gear_tactical_blade_icon/
UnityClient/Assets/Art/_IncomingAI/monster_mob_scavenger_bug_portrait/
UnityClient/Assets/Art/_IncomingAI/bg_workshop_day/
```

目录语义：

* `raw/`：AI 原始输出，允许多张候选。
* `processed/`：按 `Spec` 调整后的候选。
* `selected/`：人工或脚本初筛出的当前候选。
* `contact_sheet/`：缩略对比图。
* `manifest_snapshot.json`：生成时该 `VisualID` 对应的 Manifest entry 快照。
* `generation.json`：生成服务、模型、种子、`BatchID`、输出文件和错误记录。
* `notes.md`：人工备注、筛选理由和返工意见。

`BatchID` 仍然保留，但只作为字段，不作为目录层级。一次批量跑图可以写入多个 `VisualID` 工作区。

---

## 4. 批量生成规则

* 只读取 `Status=prompted` 的 entry。
* 每个资产至少生成 3-4 张候选。
* 每个 `VisualID` 工作区保存 `manifest_snapshot.json`，避免后续提示词变更后无法追踪。
* 每个 `VisualID` 工作区保存 `generation.json`，记录本次生成的 `BatchID`、provider、model、seed、文件路径和错误信息。
* 生成完成后填写 `BatchID`、`RawPath`，并将 `Status` 改为 `generated`。
* `RawPath` 指向 `UnityClient/Assets/Art/_IncomingAI/<VisualID>/raw`。
* `_IncomingAI/<VisualID>/raw` 禁止直接接入 Registry。

## 5. 预处理规则

预处理脚本后续建议命名：

```text
tools/美术工具/Optimize-ArtAssets.ps1
```

处理内容：

* 按结构化 `Spec.Width`、`Spec.Height`、`Spec.Format` 调整尺寸和格式。
* 按 `Spec.Background`、`Spec.AlphaRequired` 检查透明背景或不透明背景。
* 按 `Spec.SafePaddingPercent` 检查主体安全边距。
* 检查透明背景和安全边距。
* 物品、节点、义体图标裁切到 512x512。
* 怪物头像裁切到 1024x1024 或 1024x1536。
* 背景裁切到 1920x1080。
* 输出 contact sheet 供筛选。

预处理不改变主状态，只在 `_IncomingAI/<VisualID>` 内生成 `processed/` 和 `contact_sheet/`，必要时把处理结果写入 `Notes`。

---

## 6. 筛选标准

每张候选图按以下维度评估：

* 玩法识别：玩家能否立刻知道它是什么。
* 风格一致：是否符合“地底奇幻冒险 + 蒸汽朋克”的视觉方向。
* 小尺寸可读：缩小到 UI 实际尺寸后是否清楚。
* 工程可用：是否方便裁切、去底、压缩和复用。
* 内容安全：是否有水印、随机文字、签名或明显侵权特征。

不通过原因：

* 构图太满，UI 中容易被裁。
* 细节过碎，小图不可读。
* 有随机文字、签名、水印。
* 与已有素材风格冲突。
* 表达错误玩法，例如治疗道具看起来像武器。

## 7. 状态回填

| 状态 | 回填字段 |
|---|---|
| `generated` | `BatchID`、`RawPath` |
| `selected` | `SelectedPath` |
| `approved` | `ApprovedPath` |
| `registered` | `RegistryStatus=registered` |
| `validated` | 游戏内显示和基础校验通过 |
| `rejected` | `Notes` 记录废弃原因 |

Approved 路径必须与 Manifest 的 `OutputPath` 对齐。

## 8. 自动同步到 Approved

后续同步脚本建议命名：

```text
tools/美术工具/Sync-ApprovedArt.ps1
```

同步规则：

* 扫描 `UnityClient/Assets/Art/_IncomingAI/<VisualID>/`。
* 用目录名 `<VisualID>` 匹配 Manifest entry。
* 优先取 `selected/` 下按文件名升序第一张图片。
* 如果 `selected/` 为空，且脚本启用 fallback，则取 `processed/` 下按文件名升序第一张图片。
* 复制到 Manifest entry 的 `OutputPath`。
* 更新 `SelectedPath`、`ApprovedPath`，并将状态推进到 `approved`。
* 覆盖 `Approved` 中已有 PNG 时，不删除同名 `.meta` 文件。

同步脚本只处理 `_IncomingAI -> Approved`，不处理 AI 生成、预处理和 Registry 注册。
