# 资源规格与接入规范

> **定位：** 规定 Unity 工程中的美术资源目录、命名、规格和接入边界。
> **更新时间：** 2026-05-05

---

## 1. 核心原则

* 程序不引用 AI 原始产物。
* JSON 负责玩法事实，Unity 资源负责表现。
* JSON 写 `VisualID`，程序通过 `VisualAssetRegistry` 找到 Sprite、Prefab、AudioClip 或 VFX。
* Prefab 只负责表现结构，不保存物品价格、怪物 HP、掉落池等玩法数据。

---

## 2. 目录结构

初期只保留两个美术素材目录：

* `_IncomingAI`：工作区，放 AI 原始图、预处理候选、筛选记录和批次信息。
* `Approved`：正式区，放已经筛选、整理、稳定命名、可以接入 Unity 的素材。

暂不设置 `Runtime` 目录。`VisualAssetRegistry`、材质、图集等工程资产按 Unity 现有工程规范放置，不参与美术素材从生成到筛选的流转。

```text
UnityClient/Assets/Art/
  _IncomingAI/
    <VisualID>/
      raw/
        YYYYMMDD_001.png
        YYYYMMDD_002.png
      processed/
      selected/
      contact_sheet/
      notes.md
      manifest_snapshot.json
      generation.json
  Approved/
    Items/Icons/
    Monsters/Portraits/
    Nodes/Icons/
    Prosthetics/Icons/
    Chassis/
    Dolls/
    Backgrounds/
    UI/
    VFX/
    Audio/
```

`_IncomingAI` 按 `VisualID` 分类，允许临时命名和废稿；`Approved` 必须稳定命名，并与 Manifest 的 `OutputPath` 对齐。`BatchID` 不作为目录层级，只写入 Manifest 和 `generation.json`。

### 2.1 `_IncomingAI`

用途：

* AI 原始产出。
* 预处理候选。
* 初筛候选。
* contact sheet。
* 生成记录和 Manifest 快照。

规则：

* 一级目录名必须是 Manifest 中的 `VisualID`。
* 允许废稿和多个候选版本。
* `raw/`、`processed/`、`selected/` 内文件名可以临时，但同一目录内必须能按文件名升序稳定排序。
* 推荐文件名：`YYYYMMDD_001.png`、`YYYYMMDD_002.png`。
* `manifest_snapshot.json` 保存生成时该资产的 Manifest entry 快照。
* `generation.json` 保存生成服务、模型、种子、批次 ID、输出文件和错误记录。
* 程序、Prefab、Registry 禁止引用这里。

### 2.2 `Approved`

用途：

* 已筛选通过。
* 已按规格裁切、缩放、去底或统一背景。
* 已稳定命名。
* 准备进入 `VisualAssetRegistry` 的正式素材。

规则：

* 文件名必须稳定。
* 路径必须与 Manifest 的 `OutputPath` 对齐。
* Unity 表现层只引用这里的素材，或引用由这些素材制作出来的 Prefab。

### 2.3 自动同步规则

后续同步脚本建议命名：

```text
tools/美术工具/Sync-ApprovedArt.ps1
```

同步规则：

* 扫描 `UnityClient/Assets/Art/_IncomingAI/<VisualID>/`。
* 用目录名 `<VisualID>` 匹配 Manifest entry。
* 优先取 `selected/` 下按文件名升序的第一张图片。
* 如果 `selected/` 为空，且脚本显式启用 fallback，则取 `processed/` 下按文件名升序的第一张图片。
* 复制到该 entry 的 `OutputPath`。
* 更新 `SelectedPath`、`ApprovedPath` 和 `Status=approved`。
* 覆盖已有 PNG 时保留 Unity 已生成的 `.meta` 文件。

同步脚本不负责 AI 生成、不负责预处理、不负责 Registry 注册。

### 2.4 版本管理建议

版本库只追踪正式资产，不追踪工作区过程文件。

规则：

* `UnityClient/Assets/Art/Approved/**` 进入 Git。
* `Approved` 下的二进制资源建议走 `Git LFS`。
* `Approved` 下的 Unity `.meta` 文件继续走普通 Git，不进 LFS。
* `UnityClient/Assets/Art/_IncomingAI/**` 不进入 Git。
* `_IncomingAI` 下的 `raw`、`processed`、`selected`、`contact_sheet`、`generation.json`、`process_report.json` 都视为工作过程文件。

这样可以保留正式版本、减少仓库膨胀，并避免 AI 候选图和临时中间产物污染 Git 历史。

## 3. 命名规范

推荐格式：

```text
<domain>_<config_id_or_name>_<usage>.<ext>
```

示例：

```text
item_gear_tactical_blade_icon.png
monster_mob_scavenger_bug_portrait.png
node_safe_room_icon.png
bg_workshop_day.png
ui_missing_sprite.png
vfx_item_repair_use.prefab
sfx_item_repair_use.wav
```

禁止：

```text
最终版.png
长刀好看版2.png
ChatGPT Image 2026-05-05.png
repair_new_new_final.png
```

## 4. 素材规格

| 类别 | 格式 | 尺寸 | 背景 | 用途 |
|---|---|---|---|---|
| 物品图标 | PNG | 512x512 | 透明 | 背包、战利品、工坊列表、结算 |
| 怪物头像 | PNG | 1024x1024 或 1024x1536 | 透明或统一暗底 | 战斗敌人卡片、地牢预览 |
| 节点图标 | PNG | 512x512 | 透明 | 地图节点按钮 |
| 义体图标 | PNG | 512x512 | 透明 | 工坊制造、义体槽 |
| 魔偶立绘 | PNG | 1024x1536 | 透明 | 工坊展示、战斗站位 |
| 背景图 | PNG | 1920x1080 | 不限 | 工坊、战斗、地图 |
| VFX | Prefab | 按表现需要 | 透明/粒子 | 使用、命中、护盾、污染 |
| SFX | WAV/OGG | 短音效 | - | 使用、命中、UI 反馈 |

物品图标要求主体居中，留 8%-12% 安全区，64x64 仍能识别类别。

## 5. Unity 导入建议

Sprite：

* Texture Type：`Sprite (2D and UI)`
* Sprite Mode：`Single`
* Alpha Is Transparency：开启
* Max Size：图标 512，头像 1024 或 2048
* Compression：MVP 可用 `Normal Quality`
* Filter Mode：UI 图标优先 `Bilinear`

Prefab：

* 根节点命名稳定，例如 `MonsterVisual_<MonsterID>`。
* 可包含 `SpriteRenderer`、UGUI `Image`、Animator、受击点、特效挂点。
* 不包含玩法数值、掉落、背包占格、道具效果。

UI：

* 背景、装饰图、标题文本默认不阻挡射线。
* 背包拖拽、敌人选择、节点点击区域优先保证交互。

---

## 6. 接入流程

1. AI 或美术产出素材到 `_IncomingAI`。
2. 在 `_IncomingAI` 内完成预处理、缩略图和初筛。
3. 同步脚本从 `_IncomingAI/<VisualID>/selected` 或 fallback 的 `processed` 取当前图，按 Manifest 的 `OutputPath` 整理到 `Approved`。
4. 程序在 `VisualAssetRegistry` 中登记 `VisualID -> Approved Asset`。
5. UI 或表现层通过 `VisualID` 获取资源。
6. 校验缺失、重复、类型不匹配和射线遮挡。
