# AI 图片网关接入方案

> **定位：** 规定 Project P3 如何把 `tools/ai-image-gateway` 接入美术流水线，实现从 Manifest 批量跑图到 `_IncomingAI/<VisualID>` 的自动落盘和状态回填。
> **更新时间：** 2026-05-06

---

## 1. 接入目标

`tools/ai-image-gateway` 是通用图片生成层，已经接入 NovelAI。Project P3 不直接改 gateway 本体，而是在 `tools/美术工具` 中增加项目适配脚本：

```text
tools/美术工具/Run-ArtGeneration.ps1
tools/美术工具/run_art_generation.py
```

适配脚本负责：

* 读取 `美术文档/_generated/art_manifest.json`。
* 筛选 `Status=prompted` 的资产。
* 将 `PromptEN`、`NegativePromptEN`、结构化 `Spec` 转成 gateway 的 `GenerateRequest`。
* 调用 `ai_image_gateway.ImageService` 批量生成图片。
* 按 `_IncomingAI/<VisualID>` 工作区落盘。
* 写入 `generation.json` 和 `manifest_snapshot.json`。
* 回填 Manifest 的 `Status=generated`、`BatchID`、`RawPath`、`Notes`。

适配脚本不负责生成提示词、图片预处理、人工筛选、同步到 `Approved` 和 Registry 注册。

---

## 2. 依赖关系

当前 gateway 入口：

```python
from ai_image_gateway import ImageService, GenerateRequest
```

推荐运行方式：

```powershell
python -m pip install -e tools/ai-image-gateway
```

如果不安装 editable 包，`run_art_generation.py` 可以把 `tools/ai-image-gateway` 临时加入 `sys.path`。初期建议脚本自动处理 `sys.path`，降低使用门槛。

---

## 3. 配置文件

项目侧新增配置模板：

```text
tools/美术工具/ai_image_gateway.example.yaml
```

用户本地复制为：

```text
tools/美术工具/ai_image_gateway.local.yaml
```

`local` 文件不应提交。凭证统一用环境变量：

```yaml
default_provider:
  generate: novelai
  inpaint: mock
  upscale: mock

providers:
  novelai:
    enabled: true
    auth:
      access_token: ${NAI_ACCESS_TOKEN}
    settings:
      model: nai-diffusion-4-5-full
      sampler: k_euler
      scheduler: native
      steps: 28
      cfg: 5.0
      smea: none
      timeout: 120
      retry: 3
      limit_opus_free: true

logging:
  level: INFO
```

NovelAI provider 会把请求尺寸修正为 64 的倍数；在 `limit_opus_free=true` 时，大图可能被限制到约 1MP。脚本必须在 `generation.json` 中记录实际输出尺寸，不能假设 raw 图尺寸等于 Manifest 的 `Spec.Width/Height`。

## 4. 输入端

每个可生成 entry 必须具备：

| 字段 | 用途 |
|---|---|
| `VisualID` | `_IncomingAI/<VisualID>` 工作区目录名。 |
| `Domain` | 过滤、日志和统计。 |
| `PromptEN` | 正向提示词。 |
| `NegativePromptEN` | 负面提示词。 |
| `Spec.Width` | 请求宽度。 |
| `Spec.Height` | 请求高度。 |
| `Spec.Format` | 输出格式。 |
| `Status` | 只处理 `prompted`。 |

PowerShell 包装器建议支持：

```powershell
.\tools\美术工具\Run-ArtGeneration.ps1 `
  -Config tools/美术工具/ai_image_gateway.local.yaml `
  -Manifest 美术文档/_generated/art_manifest.json `
  -Provider novelai `
  -Domain item `
  -Limit 5 `
  -Variants 4 `
  -Concurrency 1 `
  -DelaySeconds 1 `
  -DryRun
```

参数建议：

| 参数 | 说明 |
|---|---|
| `-Config` | gateway 配置文件。 |
| `-Manifest` | Manifest 路径。 |
| `-Provider` | 覆盖 provider，默认走 gateway 配置。 |
| `-Domain` | 按领域过滤。 |
| `-VisualID` | 显式指定单个或多个资产。 |
| `-Priority` | 按 P0/P1 过滤。 |
| `-Limit` | 限制处理条目数。 |
| `-Variants` | 每个资产生成张数，默认 4。 |
| `-Seed` | 可选基础 seed。 |
| `-Concurrency` | 并发数，NovelAI 初期建议 1。 |
| `-DelaySeconds` | 每张图之间的请求间隔，默认 1 秒。 |
| `-BatchID` | 手工指定批次 ID。 |
| `-DryRun` | 只打印计划，不调用 gateway。 |
| `-Overwrite` | 是否允许覆盖已有 raw 输出。 |

## 5. 请求映射

Manifest entry 到 gateway 请求：

```python
GenerateRequest(
    prompt=entry["PromptEN"],
    negative_prompt=entry["NegativePromptEN"],
    width=entry["Spec"]["Width"],
    height=entry["Spec"]["Height"],
    count=1,
    seed=resolved_seed,
    provider=provider,
    output_format=entry["Spec"]["Format"],
    extra={
        "domain": entry["Domain"],
        "visual_id": entry["VisualID"],
        "asset_type": entry["AssetType"],
    },
)
```

`Variants` 不直接传给 NovelAI 的 `count`。项目适配脚本必须把每个候选拆成一次 `count=1` 请求，并在单图请求之间按 `DelaySeconds` 等待，避免 NovelAI rate limit。

Provider 特有参数不要写回 Manifest；如需覆盖模型、采样器、步数、CFG，优先写在 gateway 配置文件中，或由命令参数透传到 `extra`。

## 6. 输出端

每个资产固定输出到：

```text
UnityClient/Assets/Art/_IncomingAI/<VisualID>/
  raw/
    YYYYMMDD_001.png
    YYYYMMDD_002.png
    YYYYMMDD_003.png
    YYYYMMDD_004.png
  processed/
  selected/
  contact_sheet/
  manifest_snapshot.json
  generation.json
  notes.md
```

文件命名规则：

* 目录名负责资产归属，必须等于 `VisualID`。
* raw 文件按文件名升序稳定排序。
* 推荐格式：`YYYYMMDD_001.png`。
* 如果同一天多次生成且未开启 `Overwrite`，从现有最大序号继续递增。

---

## 7. generation.json 结构

每个 `VisualID` 工作区写一份 `generation.json`：

```json
{
  "VisualID": "item_gear_tactical_blade_icon",
  "BatchID": "ai_20260506_143000_novelai",
  "RequestID": "gateway-request-id",
  "Provider": "novelai",
  "Model": "nai-diffusion-4-5-full",
  "PromptEN": "...",
  "NegativePromptEN": "...",
  "Spec": {
    "Format": "png",
    "Width": 512,
    "Height": 512
  },
  "Requested": {
    "Width": 512,
    "Height": 512,
    "Count": 4
  },
  "Outputs": [
    {
      "File": "raw/20260506_001.png",
      "RepoPath": "UnityClient/Assets/Art/_IncomingAI/item_gear_tactical_blade_icon/raw/20260506_001.png",
      "Seed": 123456,
      "Width": 512,
      "Height": 512,
      "Params": {}
    }
  ],
  "Errors": [],
  "CreatedAt": "2026-05-06T14:30:00+08:00"
}
```

`Outputs[].Width/Height` 记录实际图片尺寸。后续预处理脚本以 Manifest 的 `Spec` 为最终目标，以 raw 实际尺寸为输入事实。

## 8. Manifest 回填

生成成功至少 1 张时：

```json
{
  "Status": "generated",
  "BatchID": "ai_20260506_143000_novelai",
  "RawPath": "UnityClient/Assets/Art/_IncomingAI/item_gear_tactical_blade_icon/raw"
}
```

如果部分失败：

* 成功张数大于 0：状态仍可进入 `generated`，`Notes` 记录错误。
* 成功张数为 0：状态保持 `prompted`，`Notes` 记录错误和时间。

脚本不得改写：

* `PromptCN`
* `PromptEN`
* `NegativePromptEN`
* `Spec`
* `OutputPath`
* `ApprovedPath`
* `RegistryStatus`

## 9. 错误处理

必须处理：

* 配置文件不存在。
* provider 未启用或凭证缺失。
* Manifest JSON 解析失败。
* entry 缺少 Prompt 或 Spec 字段。
* gateway 返回 0 张图。
* 单个资产失败不阻断整个批次。
* raw 目标文件已存在且未开启 `Overwrite`。
* 输出目录创建失败。

建议输出：

```text
[PLAN] prompted entries: 12, selected: 5, provider: novelai, variants: 4
[GENERATE] item_gear_tactical_blade_icon 1/5
[OK] item_gear_tactical_blade_icon raw=4 cost=0
[WARN] bg_workshop_day raw=0 errors=...
[DONE] success_entries=4 failed_entries=1 images=16
```

## 10. 开发拆解

### Phase 1：最小跑通

* 新增 `tools/美术工具/Run-ArtGeneration.ps1`。
* 新增 `tools/美术工具/run_art_generation.py`。
* 支持读取 Manifest、过滤 `Status=prompted`、调用 gateway mock provider。
* 输出 `_IncomingAI/<VisualID>/raw`。
* 写 `generation.json` 和 `manifest_snapshot.json`。
* 回填 Manifest。

### Phase 2：NovelAI 实接

* 接入 `ai_image_gateway.local.yaml`。
* 支持 `-Provider novelai`。
* 支持 `-Variants`、`-Limit`、`-Domain`、`-VisualID`。
* 记录实际尺寸、seed、model、generation params。

### Phase 3：稳定性

* 支持 `-DryRun`。
* 支持失败续跑。
* 支持 `-Overwrite`。
* 支持批次统计报告。
* 增加 pytest 单元测试，使用 mock provider，不调用真实 NovelAI。

### Phase 4：与后续流程衔接

* `Optimize-ArtAssets.ps1` 读取 `_IncomingAI/<VisualID>/raw` 输出 `processed` 和 `contact_sheet`。
* `Sync-ApprovedArt.ps1` 从 `selected` 或 fallback 的 `processed` 同步到 Manifest 的 `OutputPath`。

---

## 11. 验收标准

* `-DryRun` 能列出将要生成的 `VisualID`、尺寸、张数和输出目录。
* mock provider 下可无凭证生成测试图。
* NovelAI 配置正确时可生成真实图片。
* 每个成功资产都有：
  * `raw/*.png`
  * `generation.json`
  * `manifest_snapshot.json`
* Manifest 正确回填 `Status=generated`、`BatchID`、`RawPath`。
* 失败资产不破坏原有提示词和规格。
* 同一资产多次生成不会覆盖旧图，除非显式 `-Overwrite`。
