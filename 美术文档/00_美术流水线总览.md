# 美术流水线总览

> **定位：** 说明美术侧如何从配置表和预置需求发现资产、生成提示词、批量出图、预处理、筛选并接入 Unity。
> **更新时间：** 2026-05-05

---

## 1. 基本判断

程序侧的 `JSON Config -> VisualID -> VisualAssetRegistry -> Unity Asset` 方案作为运行时接入契约采用。美术侧负责它前面的生产链路：

```text
配置表 / 推导需求 / 预置需求
  -> Manifest
  -> AI 绘图提示词
  -> 批量生成
  -> 预处理
  -> 筛选
  -> Approved 素材
  -> Registry 登记
  -> 游戏内验证
```

玩法 JSON 只保存玩法事实和稳定 `VisualID`。Manifest 保存美术生产过程。`VisualAssetRegistry` 保存 Unity 资源引用。

## 3. 流程步骤

### Step 1：扫描并更新 Manifest

执行：

```powershell
.\tools\美术工具\Update-ArtManifest.ps1
```

输出：

```text
美术文档/_generated/art_manifest.json
美术文档/_generated/视觉资产Manifest.md
```

Step 1 只做资产需求发现，不写正式提示词、负面提示词和尺寸规格。

来源分三类：

* `config`：配置表直接扫出，例如 `Items/*.json -> item icon`。
* `derived`：由配置表推导，例如 `Dungeon NodePool -> node icon`。
* `preset`：预置美术需求，例如工坊背景、缺失占位图。

### Step 2：逐项补全提示词

读取 `Status=todo` 的 Manifest entry，逐项填写：

* `PromptCN`
* `PromptEN`
* `NegativePromptEN`
* 结构化 `Spec`

完成后将 `Status` 改为 `prompted`。

### Step 3：批量生成图片

后续脚本读取 `Status=prompted` 的 entry，按 `Domain`、优先级或显式列表批量出图。每个资产至少保留 3-4 张候选。

输出建议：

```text
UnityClient/Assets/Art/_IncomingAI/<VisualID>/
  raw/
    YYYYMMDD_001.png
    YYYYMMDD_002.png
  processed/
  selected/
  contact_sheet/
  manifest_snapshot.json
  generation.json
  notes.md
```

`BatchID` 只作为 Manifest 字段和 `generation.json` 记录存在，不作为 `_IncomingAI` 的目录层级。

### Step 4：图片预处理

按 `Spec` 做尺寸、裁切、透明背景、安全边距和缩略图/contact sheet。预处理结果仍留在 `_IncomingAI`，不直接进入 Registry。

### Step 5：筛选与接入

筛选通过后进入 `Approved`，然后登记 `VisualAssetRegistry`，最后游戏内验证。同步脚本按 `_IncomingAI/<VisualID>/selected` 的第一张图同步到 Manifest 的 `OutputPath`；如果 `selected` 为空且脚本启用 fallback，可取 `processed` 第一张作为当前 Approved 素材。状态流转：

```text
todo -> prompted -> generated -> selected -> approved -> registered -> validated
```

废弃项使用 `rejected`，配置移除项使用 `deprecated`。

## 4. 当前 MVP 优先级

P0：

* 物品图标
* 怪物头像
* 地图节点图标
* 缺失占位图

P1：

* 工坊背景、战斗背景、深渊路线图背景
* 深渊层级背景基调
* 魔偶立绘
* 义体图标
* 底盘表现

P2/P3：

* 怪物战斗 Prefab
* VFX/SFX
* 稀有度边框和 UI 皮肤
