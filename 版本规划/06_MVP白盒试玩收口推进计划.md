# MVP 白盒试玩收口推进计划

> **更新时间：** 2026-05-10  
> **定位：** 本文档承接当前项目状态，明确从“系统原型可运行”推进到“MVP 白盒试玩可验证”的最后收口路线。目标不是继续扩系统，而是让关键奖励、局外成长、深渊压力和表现反馈稳定发生。

## 1. 当前判断

当前项目已经完成 MVP 的主体系统骨架：数据加载、背包网格、物品拖拽、手动回合制战斗、深渊节点、战斗掉落拾取、小镇卖出、撤离/战败结算、消耗品使用、视觉资源 Registry 与自动化回归。

现在的核心风险不再是“系统能不能跑”，而是：

*   关键成长奖励可能被随机掉落截断。
*   局外养成有后端能力，但玩家未必能在 UI 和下一场战斗里明确感知。
*   2 层怪物配置了特色干涉，但运行时压迫感还不够稳定。
*   美术资源已经入库，但除物品图标外，大部分还没有被界面消费。

因此，MVP 剩余工作应按“玩法验证优先，表现增强其次”的顺序推进。

## 2. 收口目标

MVP 白盒试玩需要验证的不是内容数量，而是以下 4 个判断是否成立：

*   **背包取舍成立：** 玩家会因为格子不够而在装备、消耗品、战利品、核心材料之间做选择。
*   **局外成长成立：** 玩家带回资源后，可以升级底盘或制造义体，并在下一次下潜中明显变强。
*   **深渊压力成立：** 2 层不是简单数值膨胀，而是通过毒性、诅咒、减伤等机制破坏玩家计划。
*   **经济回流成立：** 出售战利品可以换金币，但玩家不能无脑一键卖掉成长材料。

## 3. P0：闭环脊梁

P0 完成后，项目应具备一次完整白盒试玩的最低条件。

### 3.1 Boss 保底掉落

**目标：** 关键成长材料稳定送达，不被随机掉落截断。

实施内容：

*   新增 `RewardSystem`，统一负责奖励表解析。
*   新增 `/Rewards` 奖励表，支持 `Guaranteed` 保底奖励与 `WeightedPools` 权重奖励。
*   怪物、节点、事件等调用方只配置 `RewardID`，不再直接维护掉落权重。
*   `CombatNode` 胜利结算通过怪物 `RewardID` 请求奖励，并合并为同一个拾取结果。
*   `reward_monster_elite_scrap_guard` 保底生成 `mat_core_tier1`。
*   2 层 Boss 奖励表可提前加入 `mat_core_tier2`，作为下一阶段成长钩子。
*   战利品拾取面板仍要求玩家手动塞入背包。

验收：

*   击败 1 层 Boss 后，每次都出现一阶动力核心。
*   核心塞不下时，玩家必须整理或丢弃其他物品。
*   自动化测试覆盖奖励表保底掉落和权重掉落共存。
*   旧 `LootPool` 仅作为迁移期 fallback，新配置使用 `RewardID`。

### 3.2 出售规则保护

**目标：** 让“卖钱 vs 留材料”成为决策，而不是误操作。

实施内容：

*   增加可售判定，优先支持 `CanSell` 显式字段。
*   `SellAll` 默认只出售纯卖钱战利品。
*   `Material`、`CoreMaterial`、`Cursed`、`QuestItem` 默认不可一键出售。
*   小镇 UI 对不可售物品显示原因或禁用按钮。

验收：

*   一键卖出不会移除一阶动力核心。
*   一键卖出不会移除义体制造材料。
*   纯卖钱战利品可以正常变现。

### 3.3 义体制造与生效链路

**目标：** 局外养成明确改变下一场战斗或续航。

实施内容：

*   义体配置统一为 `Effects: EffectData[]`，不再保留 `PassiveEffect` / `PassiveEffects` 兼容分支。
*   确认 `DamageMultiplier` 可由义体对指定标签武器生效。
*   实现 `RestoreSANOnCombatEnd`。
*   工坊 UI 增加制造/装备义体入口，调用 `CraftAndEquipProsthetic`。
*   制造后刷新金币、材料、已装备义体和战斗派生数值提示。

验收：

*   动力臂装备后，近战武器实际伤害提高。
*   稳压散热装备后，战斗胜利结算恢复 SAN。
*   玩家能在工坊界面制造并装备两个 MVP 义体。

## 4. P1：深渊压力

P1 完成后，2 层体验应从“更高数值”变成“规则更恶心”。

### 4.1 怪物网格干涉

实施内容：

*   `ReduceDamage`：酸液软体触发时，使玩家随机一把武器短期减伤。
*   `AddCursedItem`：畸变融合体触发时，向玩家背包塞入诅咒废件。
*   诅咒废件至少具备不可售、不可普通丢弃、或结算前强占格的一项规则。

验收：

*   2 层玩家能明显感到输出计划被打断。
*   精英怪能制造背包空间压力。

### 4.2 Toxic 携带代价

实施内容：

*   进入新节点时扫描背包内 `Toxic` 物品。
*   每件 Toxic 物品追加 SAN 消耗，MVP 默认每节点 -1 SAN。
*   地图 UI 或日志标明额外消耗来源。

验收：

*   携带污染滤芯时，进入节点 SAN 消耗高于基础消耗。
*   玩家能在高价值战利品和续航压力之间做取舍。

### 4.3 安全区成本

实施内容：

*   安全区不再只是免费恢复点。
*   MVP 可先采用“安全区只提供消耗品使用与撤离，不提供免费全恢复”的临时规则。
*   若要保留休息，则需要明确成本：时间、金币、收益倍率降低三选一。

验收：

*   玩家不会无脑选择安全区。
*   安全区成为继续深入、撤离、消耗品回血之间的决策点。

## 5. P2：表现接线

P2 不阻断规则试玩，但会显著提高可读性和反馈质量。

实施内容：

*   战斗敌人卡片读取 `PortraitID` 或默认 `monster_{MonsterID}_portrait`。
*   深渊地图节点读取 `node_combat_icon`、`node_safe_room_icon`、`node_boss_icon`。
*   工坊、深渊地图、战斗界面接入对应背景图。
*   怪物 JSON 补充 `PortraitID`，必要时补 `CombatVisualID`。

验收：

*   物品、怪物、节点、背景都能通过 `VisualAssetRegistry` 查到 Sprite。
*   缺失资源仍显示 `ui_missing_sprite`，不阻断流程。

## 6. 白盒试玩验收口径

一次 MVP 白盒试玩建议按以下路线验收：

1.  使用正式 MVP 初始装进入 1 层。
2.  经历至少 1 个普通战斗节点和 1 个安全区/路线选择。
3.  击败 1 层 Boss，出现一阶动力核心。
4.  玩家手动在战利品和装备之间整理背包，成功撤离。
5.  回工坊出售纯战利品，保留核心和材料。
6.  升级底盘或制造至少 1 个义体。
7.  进入 2 层，验证更大背包或义体效果能改变战斗。
8.  遇到 Toxic 或怪物干涉，确认深渊压力可感知。
9.  完成撤离或战败结算，确认物品流向正确。

## 7. 建议推进顺序

1.  `RewardSystem`、`/Rewards` 奖励表与 Boss 保底掉落。
2.  `CanSell` / 材料保护 / 小镇 UI 提示。
3.  义体 `Effects` 生效链路与工坊 UI。
4.  `RestoreSANOnCombatEnd` 与义体回归测试。
5.  正式 MVP 初始装与测试初始装拆分。
6.  `ReduceDamage` 与 `AddCursedItem`。
7.  `Toxic` 携带 SAN 代价。
8.  安全区成本收口。
9.  怪物头像、节点图标、背景图接线。

完成 1-5 后，可以开始第一轮白盒试玩；完成 6-9 后，再进入第二轮体验调优。
