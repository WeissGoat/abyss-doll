# MVP 客户端开发里程碑与节点规划

> **定位说明：** 本文档是客户端（Unity + C#）正式进入开发阶段的“施工图纸”。
> **开发原则：** 严格遵循“自底向上、数据先行、表现延后”的后端转客户端开发思维。每一个里程碑都必须能在不依赖花哨 UI 的情况下，通过控制台（Debug.Log）或单元测试跑通闭环。
> **必读文档:** `版本规划\01_MVP美术与UI需求清单`
---

## 🏆 阶段一：数据地基与配置连通 (Data & Config)
**目标：** 让 Unity 引擎能够正确认识我们定义的“万物实体”，并成功把策划写的 25 个 JSON 配置表全数读入内存。

*   **节点 1.1：编写纯 C# 数据容器**
    *   建立 `ItemEntity.cs` 及其核心组件（`ItemGridComponent`, `ItemCombatComponent`, `EffectData`）。
    *   建立 `DollEntity.cs` 及其组件（状态、三维、底盘 `ChassisComponent`）。
    *   建立 `PlayerProfile.cs`（全局资产与大仓库）。
*   **节点 1.2：引入 JSON 解析库**
    *   在 Unity 中引入 `Newtonsoft.Json`。
*   **节点 1.3：编写 `ConfigManager` (配置中心)**
    *   实现读取 `StreamingAssets/Configs` 目录下所有 JSON 文件的逻辑。
    *   **验收标准：** 在 Unity 启动时打印日志，能成功输出“已加载 12 个 Item，4 个 Monster...”，且字段反序列化无误。

## 🏆 阶段二：核心算法与效果引擎 (Grid & Effect Framework)
**目标：** 实现全游戏最硬核的算法逻辑——“背包俄罗斯方块”与“连结拼图效果”。

*   **节点 2.1：实现 `BackpackGrid` 容器**
    *   完成二维数组 `string[,]` 的初始化（基于底盘的死格遮罩）。
    *   完成核心难点：形状旋转（矩阵变换）与边界/碰撞检测（`CanPlaceItem`）。
*   **节点 2.2：搭建 `EffectFactory` (效果工厂)**
    *   实现 `EffectBase` 基类。
    *   编写第一个实际效果：`DamageMultiplierEffect` (增伤光环)。
    *   编写 `AddRuntimeShieldEffect` (叠盾效果)。
*   **节点 2.3：实现 `GridSolver` (网格解算器)**
    *   完成网格拓扑扫描：当物品放入包中时，能够准确找到其“右侧”或“周围”的物品，并应用 Effect。
    *   **验收标准：** 写一段测试代码，硬编码塞入一把剑和一个增伤核心，输出日志确认剑的 `RuntimeDamage` 成功增加了 30%。

## 🏆 阶段三：战斗状态机与阵营博弈 (Combat State Machine)
**目标：** 跑通基于 AP 和阵营的手动回合制战斗逻辑（依然不需要 UI，纯逻辑跑通）。

*   **节点 3.1：封装 `FighterEntity` 与 `CombatFaction`**
    *   实现 `DollFighter` 和 `MonsterFighter`，分离底层数据与战斗临时数据（`RuntimeHP`, `RuntimeShield`）。
    *   实现阵营的存活检测（`IsWipedOut`）与回合初/回合末钩子。
*   **节点 3.2：搭建 `CombatSystem` (战斗总控)**
    *   实现状态机：`PlayerTurn` -> `EnemyTurn` -> `End`。
    *   实现 AP 扣除逻辑与怪物 AI 的简易索敌攻击逻辑。
*   **节点 3.3：深渊节点树 `DungeonManager`**
    *   实现 `NodeBase`，跑通“扣除移动税 SAN -> 触发遭遇战 -> 胜利掉落”的流程。
    *   **验收标准：** 纯代码模拟一场战斗，控制台输出“玩家消耗1AP挥剑 -> 怪物扣除15血 -> 玩家结束回合 -> 怪物攻击 -> 玩家扣除10护盾”。

## 🏆 阶段四：表现层解耦与事件总线 (View & EventBus)
**目标：** 开始处理 Unity 的画面表现，用松耦合的方式把 UI 和底层的黑盒绑在一起。

*   **节点 4.1：实现 `EventBus` (全局事件中心)**
    *   定义核心事件：`OnItemPlacedEvent`, `OnHPChangedEvent`, `OnAPChangedEvent`, `OnTurnChangedEvent`。
*   **节点 4.2：实现 `GameRoot` (唯一入口)**
    *   在场景中挂载 `GameRoot`，实例化 `CoreBackend` 黑盒，并在 `Update` 中驱动 `Tick`。
*   **节点 4.3：常规 UI 搭设 (UI Toolkit / UGUI)**
    *   制作简单的左侧血条、SAN条、AP 水晶槽（监听事件自动更新）。
    *   制作右侧的 5x5 网格背景（使用 UGUI 动态生成）。
*   **节点 4.4：拖拽交互 (Drag & Drop)**
    *   实现物品色块的拖拽，调用后端的 `CanPlaceItem`，成功则触发 `EventBus` 更新画面，失败则弹回。
    *   **验收标准：** 玩家能在屏幕上把【充能手枪】拖进背包，看到旁边武器发光；点击武器，AP水晶减少一颗。

## 🏆 阶段五：系统缝合与 MVP 闭环体验 (The Loop)
**目标：** 把局外的工坊养成和局内的深渊战斗彻底缝合，形成闭环。

*   **节点 5.1：实现 `WorkshopSystem`**
    *   打通升级底盘（网格变大）和制造义体的扣除逻辑。
*   **节点 5.2：战利品入库与死亡结算**
    *   战斗胜利后，将背包中的物品实体转移到 `StashInventory`（大仓库）。
    *   战败时，清空背包，保留安全盒物品。
*   **节点 5.3：MVP 白盒试玩**
    *   **终极验收：** 完整体验一遍《01_数值模型沙盘推演方法论》中的流程。打不过1层Boss -> 读档/死回家 -> 升级底盘 -> 拿巨剑碾压一层 -> 被二层精英塞垃圾逼死。

---
**致主程：**
请严格按照此里程碑的顺序（从 1.1 到 5.3）进行编码。**在阶段三完成之前，绝对不要在 Unity 场景里花时间去调 UI 的对齐和颜色！** 保持后端的纯粹性，是我们高效推进的唯一法宝。
