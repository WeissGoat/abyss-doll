# MVP 美术与UI表现需求清单 (Art & UI Pipeline)

> **定位说明：** 既然底层的后端数据和算法（网格碰撞、工厂模式、状态机）已经敲定，这篇文档就是写给**前端UI/交互/美术**看的。
> **MVP开发原则（白盒测试法）：** 在跑通最小闭环阶段，**绝对不要等美术资源齐了再开工**。所有的图片全部用纯色方块（Whiteboxing）或免费素材代替。UI 的核心是**“交互反馈”**，而不是**“长得好看”**。

为了跑通“下深渊 → 拿素材 → 造义体/扩背包 → 战力提升再下深渊”的 MVP，我们在表现层（Unity 场景）只需要切分出 **3 个核心界面** 和 **4 类基础资产**。

---

## 一、 三大核心界面规划 (UI Scenes)

### 1. 局外：工坊整备界面 (Workshop UI)
玩家在这里进行资源消耗、底层养成和出战准备。
*   **左侧 - 魔偶展示区：**
    *   【美术资产】：一个站立的魔偶静态立绘（可以用一个矩形占位图），下面挂着 HP 和 SAN 的进度条。
    *   【UI组件】：5 个空的圆框代表“义体槽位”。如果装了【动力臂】，框里就显示一个图标。
*   **右侧 - 资源与网格区：**
    *   【UI组件 - 仓库】：一个滑动列表（Scroll View），里面平铺显示当前拥有的所有闲置物品。
    *   【UI组件 - 出战背包】：一个可视化的网格（MVP先做 4x4 和 5x5 两种规格）。玩家必须能把仓库里的东西**拖拽 (Drag & Drop)** 到这个网格里。
    *   【UI按钮】：一个硕大的【升级底盘】按钮（显示消耗1000G+核心）；一个【制造义体】按钮；一个【出发深渊】按钮。

### 2. 局内：深渊路线图界面 (Dungeon Map UI)
类似《杀戮尖塔》的预生成树状图，玩家的决策路口。
*   **主体 - 节点树：**
    *   【美术资产】：不需要画复杂的背景，只需黑底白线。
    *   【UI组件】：3 种不同的节点图标（小怪⚔️、群殴⚔️⚔️、安全区🏕️）连成线。
    *   【交互】：当前所在的节点高亮闪烁，点击下一个相连的节点，触发移动税（屏幕飘红字“-1 SAN”），随后跳转进入战斗或安全区。

### 3. 局内：核心战斗界面 (Combat UI)
“左边看戏，右边算计”的终极拼图舞台。
*   **左半屏 - 战斗舞台：**
    *   【阵营排布】：左边站着我方的魔偶，右边站着敌方怪物（支持横向排开多只，如两只拾荒虫）。
    *   【状态显示】：每个人脚下/头顶有 HP 血条，以及本回合叠出来的蓝色【护盾条 (RuntimeShield)】。怪物头顶有大大的【意图图标】（比如这回合要打你 15 点血）。
*   **右半屏 - 操作面板：**
    *   【UI组件 - 战斗背包】：展示当前的 5x5 网格。里面的武器/防具会有**高亮边框**。
    *   【UI组件 - 能量条】：显示当前回合的剩余 AP（例如 3/3 颗水晶）。
    *   【交互 - 开火】：点击背包里的武器（如短剑），扣除 1 颗 AP 水晶，播放一条弹道从武器飞向目标怪物。
    *   【UI按钮】：一个醒目的【结束回合】按钮。

## 二、 MVP 阶段极简美术资产清单 (Asset List)

MVP 阶段美术不追求细节，但必须把**尺寸和形状**卡死。

### 1. 物品图标与网格形状 (Sprites & Text Overlays)
*   因为是网格背包，所有的物品图标必须以 `1格 = 100x100 像素` 为基准。
*   **占位图规范（色块 + 文字标识）：** 纯色块在测试时极易混淆，强烈建议在色块中心加上醒目的文字（Unity中可用 TextMeshPro 挂在 Prefab 上，或直接在图上打字）。
    *   1x1 绿色方块 + 字【齿轮】（废旧齿轮）
    *   1x2 银色竖条 + 字【短剑 1AP】（生锈短剑）
    *   1x2 棕色竖条 + 字【木盾 1AP】（木盾）
    *   1x3 黑色长条 + 字【长刀 2AP】（战术长刀）
    *   2x3 红色大块 + 字【链锯 3AP】（链锯大剑）
    *   2x2 紫色方块 + 字【滤芯/毒】（污染滤芯）
    *   2x2 金色方块 + 字【核心】（一阶动力核心）

### 2. 网格交互反馈 (UI Feedback)
**这个比画画更重要！程序和UI必须把交互手感做对：**
*   **悬停高亮 (Hover)：** 鼠标抓起一把 2x3 的大剑悬停在背包上时，底下的 6 个格子要亮起绿光（代表能放）或红光（代表撞车了放不下）。
*   **连结激活 (Link Effect)：** 当把【充能手枪】放到背包里，且它右边恰好有一把武器时，手枪和右边武器的边框要同时闪烁一下金光，证明 `EffectFactory` 里的“加伤Buff”挂载成功了。
*   **受锁/污染反馈：** 怪物如果对格子施放了 `GridInterference`（如塞垃圾），那个格子上会生硬地出现一个冒着紫气（加点粒子特效）的色块，怎么拖都拖不动。

### 3. 战斗视觉反馈 (VFX & Floating Text)
不追求酷炫的技能特效，但数值变化必须清晰。
*   **飘字系统 (Damage Text)：** 红色飘字代表扣血（-15），蓝色代表加盾（+10🛡️），紫色代表掉理智（-2 SAN）。
*   **受击闪白 (Hit Flash)：** 被打的人变白闪烁 0.1 秒，代表掉血了。

## 三、 UI 技术选型与开发步调建议 (给客户端同学)

强烈建议结合 AI 辅助代码生成，采用“重代码，轻拖拽”的工作流。

### 1. UI 技术单轨制选型 (纯 UGUI)
*   **全部界面（局外工坊、战斗HUD、核心背包等）：全面统一为 `UGUI`**
    *   【架构铁律】我们在早期开发中发现，网格碰撞与拖拽（射线检测 Raycast）极易与 UI Toolkit 发生不可预知的拦截冲突。因此，项目**已彻底废弃 UI Toolkit**。
    *   所有界面均使用传统的 Canvas + UGUI 组件进行拼装，配合自动化 Editor 脚本（`MVPEditorSetup.cs`）生成场景骨架，避免手工拖拽的误差。

### 2. MVP 开发落地 4 步走

1.  **先做“死”的预制体：** 在 Unity 里只做一个极其简单的「基础格子 Prefab」和「物品色块 Prefab」。不要手动摆 25 个格子！
2.  **代码动态生成网格：** 写一个 `GridGenerator.cs` 脚本，在游戏启动时，根据底层 `CoreBackend` 的底盘数据，用 `Instantiate` 瞬间循环生成 5x5 的格子阵列。
3.  **接通底层逻辑 (事件总线)：** 写一个 `DragAndDropManager.cs`，实现拖拽色块松手时，向后端发起 `GameRoot.Core.Inventory.RequestPlaceItem()`。如果后端判定可以放，后端会抛出 `OnItemPlacedEvent` 事件，前端监听到事件后再真正把色块吸附到目标位置。
4.  **做战斗表现队列 (Visual Queue)：** 切忌在后端计算完战斗后瞬间改变血条。做一个排队系统，按顺序播放攻击动画和受击飘字，然后提供一个【结束回合】按钮触发下一轮后端状态机流转。

只要这 4 步在屏幕上能点得动，我们这个复杂的“背包搜打撤”就算是拥有灵魂了！
