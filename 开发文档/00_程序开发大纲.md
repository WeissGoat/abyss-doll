# 程序开发大纲与系统索引 (Development Outline)

> **定位说明：** 本文档是客户端程序架构的最高大纲，列出了全游戏所有需要开发的子系统模块。
> **阅读指南：** 本文档严格保持了最初设计的系统分类结构。如果某一模块已经有详细的落地 C# 实现指南，该条目下已删除旧的设计草案，直接引读至对应的详细文档；如果尚未编写详细指南的系统（如特质系统、叙事系统），则在此保留最初的架构草案与设计意图。

掌管城镇生活、资产与仓储。

*   **`PlayerProfile` (玩家账号实体)**
    *   👉 **详细代码落地指南请见：** [`01_核心数据与实体容器(CoreData).md`](./01_核心数据与实体容器(CoreData).md)
*   **`LocationSystem` (地点路由系统)**
    *   控制场景内UI或数据的切换出口。

        *   **`ShopManager` (商店系统)**：获取物品估值，包含动态物价浮动系数字典。
        *   **`GuildManager` (公会系统)**：记录不同势力的 `Reputation` (声望等级) 并解禁购买配方。

## 💬 6. 叙事与事件管线 (Narrative & Event Pipeline)
负责解耦你的剧情逻辑。

*   **`ConditionEvaluator` (条件解析器)**
    *   读取类似 `Req_Money_>_100` 或 `Req_HasTag_恐虫` 的逻辑。
*   **`EventSystem` (事件系统)**
    *   包含游戏内所有的特殊遭遇或周结事件配置。利用条件解析器裁定是否触发。
    *   核心方法：`TriggerEvent(EventID)` -> 执行相关 `EffectSystem` 的表现。
    *   *(注：UI 状态表现更新已分离至 `GameEventBus`，详见 [`05_表现层架构与事件总线(ViewAndEventBus).md`](./05_表现层架构与事件总线(ViewAndEventBus).md))*
*   **`DialogueSystem` / `ChapterSystem` (对话与篇章驱动器)**
    *   作为最高层的叙事观测者，监听 `EventSystem` 或者玩家的节点进展。
    *   调用UI系统呈现立绘、对话框和分支选项，以此实现剧情。
