# 美术资源接入规范

> **定位：** 约定 Unity 工程中美术资源的目录、命名、引用边界和导入标准，服务于 AI 素材工作流与后续正式美术接入。
> **更新时间：** 2026-05-05

---

## 1. 核心原则

### 1.1 程序不直接依赖 AI 原始产物
AI 生成素材允许快速试错，但原始产出通常会包含废稿、变体、尺寸不统一和临时命名。程序侧只接入经过筛选的资源层，不直接扫描或引用 `_IncomingAI`。

### 1.2 JSON 负责玩法事实，Unity 资源负责表现
物品尺寸、效果、价值、掉落来源、怪物数值等继续放在 JSON 配置中。图片、Prefab、音效、特效等表现资源通过稳定的视觉 ID 间接绑定。

### 1.3 Prefab 是运行时实例化单位，Registry 是资源索引契约
项目可以使用 Prefab，但 JSON 中不直接写 Unity 对象引用。JSON 写 `VisualID`，程序通过 `VisualAssetRegistry` 找到对应 Sprite、Prefab、AudioClip 或 VFX Prefab。

## 2. 推荐目录结构

```text
UnityClient/Assets/Art/
  _IncomingAI/
    2026-05-05_items_batch01/
    2026-05-05_monsters_batch01/
  Approved/
    Items/
      Icons/
      WorldPrefabs/
    Monsters/
      Portraits/
      CombatPrefabs/
    Nodes/
      Icons/
    UI/
      Frames/
      Buttons/
      Panels/
    Backgrounds/
      Workshop/
      Dungeon/
      Combat/
    VFX/
    Audio/
  Runtime/
    Registries/
    Materials/
    Atlases/
```

### 2.1 `_IncomingAI`
用于存放 AI 原始产出，可以按日期、主题、批次自由组织。这里允许临时命名和大量变体。

约束：
* 程序不引用此目录。
* 场景、Prefab、Registry 不绑定此目录中的资源。
* 大批量废稿可以不进入版本库，后续根据团队协作方式决定是否使用 Git LFS 或外部网盘。

### 2.2 `Approved`
用于存放已筛选、可进入游戏的素材。进入此目录后，需要遵守命名、尺寸、透明背景、用途分类等规范。

约束：
* 程序可以引用此目录。
* 文件名必须稳定。
* 删除、改名、替换已接入资源前，需要同步检查 Registry 和配置引用。

### 2.3 `Runtime`
用于存放程序运行时直接依赖的整理结果，例如资源注册表、图集、材质、运行时专用 Prefab。

约束：
* 这里不是原画工作区。
* 这里的资源更偏向“已接入版本”，应尽量减少临时变动。

## 3. 命名规范

统一使用小写英文、下划线和用途后缀：

```text
item_gear_tactical_blade_icon.png
item_con_repair_kit_icon.png
monster_mob_scavenger_bug_portrait.png
monster_elite_scrap_guard_combat.prefab
node_safe_room_icon.png
bg_workshop_day.png
ui_frame_item_common.png
vfx_item_repair_use.prefab
sfx_item_repair_use.wav
```

禁止使用：

```text
最终版.png
长刀好看版2.png
ChatGPT Image 2026-05-05.png
repair_new_new_final.png
```

### 3.1 推荐命名格式

```text
<domain>_<config_id_or_name>_<usage>.<ext>
```

常用 `domain`：
* `item`
* `monster`
* `node`
* `bg`
* `ui`
* `vfx`
* `sfx`

常用 `usage`：
* `icon`
* `portrait`
* `combat`
* `world`
* `frame`
* `panel`
* `button`
* `use`
* `hit`

## 4. 素材规格

### 4.1 物品图标

推荐：
* 格式：PNG
* 尺寸：512x512
* 背景：透明
* 构图：主体完整，边缘留 8%-12% 安全区
* Unity 类型：Sprite

用途：
* 背包格子
* 战利品拾取
* 工坊卖出列表
* 结算清单

### 4.2 怪物头像

推荐：
* 格式：PNG
* 尺寸：1024x1024 或 1024x1536
* 背景：透明或统一底色
* Unity 类型：Sprite

用途：
* 战斗敌人卡片
* 地牢预览
* 图鉴或调试面板

### 4.3 怪物战斗 Prefab

推荐组成：
* 根节点：`MonsterVisual_<MonsterID>`
* `SpriteRenderer` 或 UGUI `Image`
* 可选 Animator
* 可选受击点、飘字点、特效挂点

注意：
* Prefab 不保存怪物数值。
* Prefab 不决定怪物掉落。
* Prefab 只负责表现和挂点。

### 4.4 节点图标

推荐：
* 格式：PNG
* 尺寸：512x512
* 背景：透明
* 分类：Combat、SafeRoom、Boss、Event、Exit

### 4.5 背景图

推荐：
* 工坊背景：1920x1080
* 战斗背景：1920x1080
* 地图背景：1920x1080 或可平铺纹理
* Unity 类型：Sprite 或 Texture2D

MVP 阶段优先级低于物品图标和怪物头像。

## 5. Unity 导入设置建议

### 5.1 Sprite 类资源

建议：
* Texture Type：`Sprite (2D and UI)`
* Sprite Mode：`Single`
* Alpha Is Transparency：开启
* Max Size：物品图标 512，怪物头像 1024 或 2048
* Compression：MVP 阶段可用 `Normal Quality`
* Filter Mode：UI 图标优先 `Bilinear`

### 5.2 Prefab 类资源

建议：
* Prefab 只做表现结构。
* 所有玩法数据从 JSON 和运行时实体读取。
* Prefab 内的按钮、遮罩、装饰图需要检查 `raycastTarget`，避免遮挡拖拽和点击。

### 5.3 音效类资源

建议：
* 短音效使用 WAV 或 OGG。
* 命名区分 `sfx_item_xxx_use`、`sfx_weapon_xxx_hit`。
* 音量、混音组后续统一接入 AudioMixer。

## 6. 协作边界

### 6.1 美术负责
* AI 原始素材筛选。
* 将确定可用素材移动到 `Approved`。
* 按命名规范重命名。
* 标记用途，例如 icon、portrait、combat、bg。

### 6.2 程序负责
* 维护 `VisualAssetRegistry`。
* 将 `VisualID` 接入 UI 和战斗表现。
* 提供缺失资源 fallback。
* 提供配置引用校验。

### 6.3 策划或系统设计负责
* 在 JSON 中填写稳定的视觉 ID。
* 确认表现资源与玩法定位一致。
* 避免在素材文件名中表达数值事实。

---

## 7. 接入流程

1. AI 或美术产出素材，放入 `_IncomingAI`。
2. 筛选通过后，整理到 `Approved`。
3. 按规范重命名。
4. 程序在 `VisualAssetRegistry` 中登记 `VisualID -> Unity Asset`。
5. JSON 配置填写对应 `VisualID`。
6. 运行时 UI 或表现层通过 `VisualID` 获取资源。
7. 自动化校验报告缺失、重复或未使用的视觉 ID。

---

## 8. MVP 阶段优先级

第一优先级：
* 物品图标
* 怪物头像
* 节点图标

第二优先级：
* 工坊背景
* 战斗背景
* 地图背景

第三优先级：
* 角色或怪物战斗 Prefab
* 特效
* 音效
* 图集和资源打包优化

---

## 9. 风险与注意事项

### 9.1 不要出现两份真相
物品的尺寸、价格、效果、类型必须以 JSON 为准。图片和 Prefab 不应再写一份玩法数据。

### 9.2 不要让临时素材进入正式引用
临时素材可以出现在 `_IncomingAI`，但一旦被场景、Prefab 或 Registry 引用，就应该进入 `Approved` 或 `Runtime`。

### 9.3 注意 UI 射线遮挡
背包拖拽、战利品拾取、敌人选择都依赖射线检测。装饰图片、背景图、标题文本默认不应阻挡射线。
