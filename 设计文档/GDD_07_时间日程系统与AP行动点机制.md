# 详案_07：时间日程系统与天数轮转机制 (GDD_07)

> **定位说明：** 本系统是驱动整个宏观游戏循环的"节拍器"。它通过"天数（Day）"的不可逆流逝和每天唯一的核心抉择，使经营、养成与深渊探索三个模块形成真实的博弈关系。
>
> **重要变更记录：** 本文档已废弃原有的"每天3AP（行动点）分配制"，替换为"单线程天数轮转 + 零时耗营地维护"的双层行动模型。时间的基本单位从"行动点"变为"天数"，每天只有一个推进时间的核心决定。

---

## 第一部分：时间结构总览

### 1.1 时间层级

游戏时间分为三个层级，嵌套运作：

```
【月】 Month  ─ 宏观结算单位（租金/目标/结局判定）
  └─ 【周】 Week  ─ 中观节奏单位（物价刷新/委托刷新）
        └─ 【天】 Day  ─ 微观操作单位（玩家每日核心抉择）
```

| 层级 | 时长 | 主要意义 |
|---|---|---|
| 天（Day） | 玩家做出一个核心抉择的基本单位 | 深渊或休息的二选一、零时耗维护、营业结算 |
| 周（Week） | 每周末触发结算事件 | 物价波动、委托刷新、魔偶状态累积 |
| 月（Month） | 每月末触发强制结算 | 租金扣除、阶段评级、结局推进 |

### 1.2 一局游戏的总时长

一局游戏（单局单魔偶制）的总生命周期为 **12个月**（即一年），每月固定为 **4周**，每周为 **7天**。

- 总计：12 × 4 × 7 = **336天**
- 玩家目标在此期限内达成特定条件触发对应结局
- 时间耗尽时强制进入结局判定，无论玩家当时状态如何

## 第二部分：双层行动模型

### 2.1 设计核心——从"点数分配"到"非此即彼的残酷抉择"

旧的3AP系统被废弃。时间单位简化为纯粹的**"天数（Day）"**。倒计时的压迫感不变（例如：距离交租还有15天），但这15天里，玩家每天只能做一个绝对的核心决定。

在小镇的安全屋（UI主界面）中，所有操作分为两类：**零时耗的营地维护** 和 **推进次日的单选行为**。

### 2.2 零时耗行为（不推进天数，随时可做）

这些是玩家在做出核心抉择前进行的策略准备和情感安抚，无论怎么操作都不会推进天数。

| 行为 | 说明 | 消耗 |
|---|---|---|
| **变现与购物** | 将战利品扔进出货箱变现，或在商店采购消耗品 | 仅消耗金币 |
| **装备与组装** | 重新排布6×6网格背包，安装/拆卸义体 | 无 |
| **轻度日常交互** | 和魔偶闲聊、摸头、换衣服、送小礼物 | 无 / 消耗赠礼物品 |
| **势力拜访** | 向势力交付物品、接取委托、使用势力商店 | 仅消耗对应物品或金币 |
| **查看情报** | 查看小镇传闻（物价波动）、VIP商会订单 | 无 |
| **橱窗陈列** | 在VIP橱窗中排布高价值物品（网格拼图） | 无 |

> **防滥用机制**：轻度日常交互虽然免费，但只产出"好感度"和"隐藏情绪值"，或者只能把SAN值恢复到一个很低的低保线（比如最多回复至20%上限）。"我可以一直陪你聊天，但仅仅是聊天，治不好你身上的贯穿伤。"

### 2.3 推进天数的核心操作（每天二选一）

当玩家在安全屋整顿好之后，必须决定**"今天这漫长的一天，我们要用来干嘛？"**这时按下按钮，时间就会进入第二天。

#### 选择 A：跳下深渊 (Dive)

| 维度 | 内容 |
|---|---|
| **结果** | 进入核心的背包战斗探索环节 |
| **收益** | 带回大量战利品和素材 |
| **代价** | 魔偶带着满身伤痕和暴跌的SAN值回来。且距离交租日逼近一天 |

#### 选择 B：深度疗愈 / 停工休息 (Rest & Repair)

| 维度 | 内容 |
|---|---|
| **结果** | 直接跳过这一天，不进入战斗。触发特殊的双人羁绊剧情、深度入睡或手术台修理 |
| **收益** | 大幅度甚至完全恢复魔偶的SAN值和HP，清除负面特质，解锁深层好感剧情 |
| **代价** | 今天颗粒无收！房租的死线无情地逼近了一天 |

> **设计意图**：这种"非此即彼"的框架比旧的3AP分配制更加纯粹和犀利。玩家不再玩数字拼图，而是每天面对一个有情感重量的抉择——"她已经遍体鳞伤了，但再不去深渊就交不起房租了。"

## 第三部分：一天的完整流程（昼夜交替制）

### 3.1 三阶段日循环

每一天分为三个清晰的阶段，承载不同的游戏体验：

```
🌅 备战期（零时耗）→  ☀️ 核心抉择（推进天数）→  🌙 营业结算期（自动）
```

#### 🌅 备战期 — 策略与养成（小镇安全屋）

- 玩家起床，查看今天的"小镇传闻"（物价波动）和"VIP商会订单"
- 给魔偶整理装备、安装义体、整理出战背包（网格拼图）
- 进行零时耗交互：闲聊/送礼/势力拜访/采购
- **核心驱动**：制定今天的策略——要不要下深渊？带什么下去？

#### ☀️ 核心抉择 — 深渊或休息（推进天数的唯一时刻）

- 玩家按下「跳下深渊」或「停工休息」按钮
- 选择深渊 → 进入搜打撤战斗环节
- 选择休息 → 触发疗愈/羁绊剧情，时间直接推进到夜晚

#### 🌙 营业结算期 — 释放与收获

- **每天都会触发**（无论当日是否下过深渊）
- 带着战利品拆箱分拣（若当天去过深渊），或处理仓库存货（若当天休息）
- 进行三步营业流程（详见 GDD_04）
- 一键营业 → 蒙太奇动画 → 营业日报
- **核心驱动**：数钱、正向反馈、触发随机经营事件

### 3.2 典型日程示例

**【深渊搏命日】**
```
🌅 查看订单 → 整理出战背包 → 带上VIP目标物清单
☀️ 跳下深渊 → 搜打撤！极限背包博弈
🌙 满载而归 → 倾倒出货箱 → VIP橱窗拼图 → 一键营业 → 大赚一笔
```
→ 结果：金币暴涨，但魔偶SAN值暴跌、机体磨损加重

**【停工疗愈日】**
```
🌅 给魔偶换上便服 → 送一件小礼物 → 聊聊天（零时耗）
☀️ 选择休息 → 触发深度疗愈剧情/手术台修理
🌙 营业（出售仓库存货）→ 少量收入
```
→ 结果：SAN值和HP大幅恢复，好感度提升，但今天没有新战利品进账

**【情报收集日】**
```
🌅 查看传闻 → 拜访势力交付委托 → 用声望兑换新图纸
☀️ 跳下深渊 → 带着VIP订单目标针对性搜索
🌙 带回目标物 → 橱窗摆放 → 一键营业
```
→ 结果：同时推进势力声望和经济目标

## 第四部分：周结算机制

每周末（第7天结束后）自动触发周结算，无需玩家额外操作。

### 4.1 周结算内容

| 结算项目 | 说明 |
|---|---|
| **魔偶状态累积结算** | 本周侵蚀度累积量、磨损度累积量正式写入魔偶数据（局内实时数值在周末落地） |
| **物价波动刷新** | 小镇情报板推送下周传闻，各类物品基础售价系数更新 |
| **委托刷新** | 势力订单列表更新，截止期限倒计时推进 |
| **过劳判定** | 检测本周是否连续5天以上选择了深渊探索，若是则触发「疲劳积累」（见第六部分） |
| **特殊事件触发窗口** | 部分剧情事件绑定在周末触发（如好感度阶梯事件），在此时弹出 |

### 4.2 物价刷新的对玩家影响

物价刷新是告知玩家"下周应该去深渊打什么"的信号。
玩家在周结算后看到新的传闻（如「法师塔急需深渊龙血」），会在下周主动为这个目标预留背包空间，形成"情报引导行动"的正向循环。

## 第五部分：月结算与租金压力

### 5.1 月结算内容

每月第28天结束后触发月结算：

| 结算项目 | 说明 |
|---|---|
| **租金扣除** | 强制从玩家金币中扣除当月"工坊许可费/深渊许可证费"，金额随月份递增 |
| **阶段进化判定** | 检测魔偶三维属性是否达到进化阈值，触发阶位升级或结局分支推进 |
| **月度成就记录** | 记录本月下潜最深层数、总收入、好感度变化量等，作为最终评级参考数据 |
| **结局条件推进** | 检查特定结局的达成进度（详见GDD_05剧本系统） |

### 5.2 租金压力曲线

租金设计为**每月递增**，确保游戏后期经济压力持续存在，防止玩家进入"早期暴富后无所事事"的死局：

| 月份区间 | 阶段名称 | 设计意图 |
|---|---|---|
| 第1~3月 | 初始困难期 | 租金相对温和，引导玩家熟悉核心循环 |
| 第4~6月 | 追逐压力期 | 租金明显上升，玩家必须开始优化每日抉择 |
| 第7~9月 | 高压运营期 | 租金较重，需要深层深渊+高效橱窗经营双管齐下 |
| 第10~12月 | 决战倒计时 | 租金达到峰值，同时结局条件开始收口 |

> **设计意图**：租金不是惩罚，而是「强制驱动引擎」。玩家每个月底必须面对一次"账单危机感"，这种定期施压让玩家始终有下一次下深渊的动力，防止游戏节奏沉寂。

### 5.3 租金违约后果

若玩家在月末无法缴清租金：

1. **轻度违约**（欠款 < 50%）：产生"欠债记录"，下个月租金额外附加利息；小镇商人短期内对玩家提高售价警戒。
2. **重度违约**（欠款 ≥ 50%）：触发「债主登门」强制剧情事件；强制没收仓库中部分高价值物品充抵；魔偶目睹此景，好感度下降。
3. **连续两月违约**：触发游戏的「经济破产」Bad Ending，本局游戏强制结束。

## 第六部分：状态对抉择的反向约束

天数轮转系统不只是玩家单方面选择行动，**魔偶的当前状态会主动限制或惩罚某些选项**。

### 6.1 过劳状态

**触发条件**：连续5天或以上选择「跳下深渊」，没有穿插休息日。

**表现**：
- 安全屋界面中「跳下深渊」按钮变灰，附注：「她似乎已经精疲力竭，不愿意再站起来了。」
- 强行触发（若有特定强制手段）会导致恶性事件：在深渊中随机出现不受指令控制的"走错方向"
- 解除方法：安排至少 **2天连续休息日**（选择「深度疗愈」）

### 6.2 高侵蚀状态

**触发条件**：魔偶当前侵蚀度超过 70%（重度）。

**表现**：
- 安全屋界面中「跳下深渊」按钮变为橙色警告，提示："当前精神状态不稳定，深渊侵蚀可能引发幻觉。"
- 强行下深渊：局内背包会随机出现无法消除的幻觉色块，封锁 1~2 个格子
- 解除方法：选择「深度疗愈」或购买高级保养服务将侵蚀度降至安全线以下

### 6.3 机体重损状态

**触发条件**：深渊战斗失败撤离后，魔偶积累了高额磨损度（严重战损）。

**表现**：
- 安全屋界面中「跳下深渊」按钮强制锁定，无法选择，附注：「她需要先修复关节，否则无法正常运作。」
- 必须先选择「深度疗愈」或使用零时耗的基础保养将磨损度降至阈值以下，才能解锁下潜

### 6.4 好感度极低（冰点）状态

**触发条件**：好感度降至最低等级（长期压榨、多次违约、忽视保养）。

**表现**：
- 零时耗交互中的「闲聊/摸头」选项仍存在，但魔偶头像为冰冷状态，点击后提示："她转过身去，不想理你。"
- 闲聊的好感度恢复效率大幅降低（冰封难以化解）
- 营业结算时，魔偶会消极怠工，倾倒箱物品售价降低，偶尔主动赶走客人

## 第七部分：特殊时间事件

某些事件不受玩家日常抉择控制，会强制插入或改变当天/当周的流程。

### 7.1 强制触发事件（不可跳过）

| 事件名 | 触发条件 | 对流程的影响 |
|---|---|---|
| 债主登门 | 月末欠债时 | 强制占用当晚营业结算期，玩家必须响应剧情 |
| 魔偶崩溃 | 侵蚀度达到100% | 强制进入「崩溃处置」流程，当天无法选择深渊，只能处理危机 |
| 深渊异动警报 | 特定剧情节点触发 | 当周某一天出现特殊深渊任务选项（限时） |
| 神秘买家拜访 | 随机，概率较低 | 营业结算期出现高价收购机会，不响应则错过 |

### 7.2 限时窗口事件（可选择抓住或放弃）

| 事件名 | 触发条件 | 限时 | 奖励 |
|---|---|---|---|
| 集市特卖日 | 每月第2周末 | 当天 | 小镇商品8折，可购买平时买不到的稀有道具 |
| 黑市商人造访 | 随机夜晚，概率触发 | 当晚营业期 | 高价收购诅咒物品、黑市独有材料交易 |
| 势力庆典 | 每季度一次 | 当周 | 声望奖励翻倍，特殊图纸短期上架 |
| 深渊潮汐异常 | 特定月份 | 当周 | 深渊2~5层掉落率大幅提升，但侵蚀度增长速度也翻倍 |

## 第八部分：天数轮转与体验节奏设计

### 8.1 "非此即彼"是核心体验

天数轮转系统的根本设计目的不是"让玩家制定完美的策略表"，而是**持续地逼迫玩家在深渊战利品和魔偶的身心健康之间做取舍**。

每一天玩家面临的心理问题是：
- 「租金还差2000G，我该强行下深渊，还是让她先休息？」
- 「SAN值已经红了，但今天不下深渊明天仓库就断货了。」
- 「传闻说本周魔兽膏滋价格涨价，但她已经连续下了4天深渊了……」

这种抉择**没有最优解**，只有最符合当前局势的解。

### 8.2 两种极端策略的博弈结果

| 策略类型 | 核心行为 | 短期结果 | 长期风险 |
|---|---|---|---|
| **深渊激进派** | 每天下深渊，拒绝休息 | 战利品充足，金币滚滚 | 过劳+侵蚀爆表→强制锁定深渊→恶性循环崩盘 |
| **安逸养成派** | 大量时间用于休息/疗愈 | 好感度高，魔偶状态满 | 金币匮乏，月末租金危机频繁→深渊进度停滞 |

游戏要求玩家**在两极之间动态切换**，这种"不得不妥协"本身就是乐趣所在。

### 8.3 时间的重量感

时间的流逝应该在表现上给玩家真实的压迫感：

- 每次确认核心抉择后，画面有简短的"时间流动"动效（如窗外光线从晨至暮）
- 月末到来前3天，UI上显示「还有X天缴租」的倒计时标注
- 魔偶的状态立绘会随时间推移产生细微变化（经历越多越有"历程感"）
- 游戏中没有"暂停时间"或"跳过时间"的机制——玩家每天必须做一个决定，不能空过

### 8.4 为何"零时耗行为"必须免费

零时耗行为的免费设计不是偷懒，而是对核心玩法的保护：

1. **避免脑力疲劳叠加**：深渊中的背包战斗已经是极度消耗脑力的高压体验。回到小镇后如果还要为"整理背包花不花时间"而焦虑，玩家会累到弃坑。
2. **凸显核心博弈**：当所有次要操作都不需要时间成本时，"下深渊 vs 休息"的抉择才真正凸显出来——这是每天唯一有重量的决定。
3. **保护养成的纯粹性**：摸头聊天送礼不花时间，意味着玩家不需要把"陪伴魔偶"和"下深渊赚钱"做对立——它们在不同维度上运作。真正的对立只存在于"下深渊（伤害她）vs 休息（保护她但没收入）"这个残酷的轴心上。

## 第九部分：设计边界与平衡原则

### 9.1 不做的事

- **不设计"完美日程"**：不存在一套能同时满足所有需求的策略，这是故意的设计。
- **不引入"加速时间"道具**：时间是游戏中唯一不可购买、不可增加的资源，必须保持稀缺性。
- **不允许"跳过营业结算"**：每天的营业结算是强制执行的——哪怕仓库里空空如也，也会播放一段"今日无货可卖"的冷清画面，用来刺激玩家的危机感。

### 9.2 防止"无脑最优解"

若玩家试图机械重复同一套模式（如"永远下深渊，不休息"），系统通过以下手段自然打破：

- 过劳状态强制插入休息需求（连续5天深渊后强制停摆）
- 侵蚀度与磨损度的累积使深渊效率递减
- 委托截止日临近形成突发压力
- 黑市/集市等限时事件改变当天的最优选择
- 月末租金的数量波动要求玩家动态调整策略

### 9.3 新手引导期

游戏第一个月（第1~28天）的租金设计极低，核心抉择的压力也相对宽松，旨在：

- 让玩家自由探索两种选择的不同后果，发现系统规律
- 在没有"死亡压力"的环境下建立对天数博弈的直觉认知
- 通过前几天"无压力的抉择"让玩家形成情感依附，之后的压力才会真正产生代入感
