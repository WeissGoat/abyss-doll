# 详案_03：人偶实体对象与好感双轨机制 (GDD_03)

> **定位说明：** 魔偶不仅仅是一个血条与背包容器的外壳。本文档将详细解构魔偶的“羁绊（长线）”与“理智（短线）”双轨制状态引擎，以及“性格特质 (Traits) 系统”是如何改变物理法则的。

---

## 第一部分：UI 层与工坊交互逻辑（表现层）

日常的模拟养成和情感建立全部发生在主界面的【工坊 (Workshop)】中。

### 1. 工坊主体界面布局
- **视觉中心：** 魔偶居中站立（Live2D / Spine）。拥有视线跟随鼠标的功能，体现“存活感”。
- **三大核心量表：** 这是完全不同的维度，不能混为一谈。
  - **[硬件底盘]：** 金属硬壳质感的进度条，满额则解锁更大的背包网格。
  - **[理智液槽 (SAN)]：** 左侧一个类似液体管道的量表，深渊受损时液位下降并变黑发泡。
  - **[好感度之心 (Bond)]：** 右侧带有发光粒子的心脏标记，等级突破时会有专门的动画。
- **特质面板区：** 角色脚下的插槽，展示当前拥有的所有【性格标签 (Tags)】（红色为负面，金色为正面，白色为中性）。

### 2. 玩家反馈互动动作
- **点击触摸：** 点击头部/身体能够播放常规反馈对话。如果在理智槽严重不足（泛黑）时触摸，魔偶会拨开鼠标呈现抗拒动画并播放压抑语音。
- **赠礼模块 (`Drag -> Gift`)：** 玩家可以在工坊打开库存，将【非战斗类/情感类】道具直接拖拽到魔偶身上完成赠送。

## 第二部分：核心机制法则与状态机（业务逻辑层）

剥离掉各种包装外壳，魔偶在代码底层是一个**挂载了多个状态机且监听全场事件的超级实体 (Super Entity)**。

### 1. 理智值循环驱动模型 (短线生命线)
理智值（Sanity）代表现在的稳定度，通过金钱和休息可以“购买拉升”。如果拉拽失败，深渊探险将极为痛苦。
- **阈值事件钩子：**
  - 当 `CurrentSAN == MaxSAN` (100%) 时，全背包武器充能速度自带 `+10%` Buff。
  - 当 `CurrentSAN < 30%` (危险线) 时，在深渊每点开一个地图节点，不再掉 2 点，而是掉 4 点。
  - 当 `CurrentSAN == 0` 时，系统发布 `Event_Sanity_Break`，触发防逃逸的死亡法则（参考详案02）。

### 2. 羁绊阶梯与护盾（长线底蕴积累）
无论今天在深渊被虐得多惨，只要局外给了关怀，好感度**绝不倒退**。
- **好感度突破 (Bond Levels)：** 分为 `Lv.1` 到 `Lv.10`。每次突破需完成**一段指定的对话或事件**。
- **羁绊的实战体现（奇迹算法）：**
  - 这是一套游离在普通规则之外的特权代码。
  - 核心算法：当 `Damage > CurrentHP`（即将致死打击）时，如果 `Current_Bond_Level >= 5` 且当局防死标识 `HasRevived == False`。
  - 强行切断结算：`CancelDamage()` -> 弹回 1 点 HP -> 发送全屏剧烈红光闪白特效 -> `HasRevived = True`。 
  - 这是游戏想传达的“哪怕全世界物理法则都要你死，你的情感依然能创造一次奇迹”。

### 3. 性格特质反馈引擎 (Trait Engine)
让数值不仅是数值，必须让“性格”在游戏里**实质性地捣乱**或**帮忙**。
特质（Trait）本质上是一个**事件监听器组合**。
- **结构定义：** 一条具体的 Trait 包含 `[触发时机]` + `[条件验证]` + `[执行逻辑]`。
- 比如，魔偶获得了一个因为在深渊频繁断手而造成的心理阴影特质 【近战恐惧】。
  - **触发时机：** `On_Item_Placed_In_Bag` (向包里放物品时)。
  - **条件验证：** 玩家拖进去的是不是带 `[Melee]` 标签的物品？
  - **执行逻辑：** 如果是，强制魔偶大喊一声语音“离我远点”，并立刻扣除 5 点理智值。如果理智值不够，直接把这把刀从包里弹出去（拒绝装备）。

## 第三部分：原型样例库 (MVP Demos)

### 1. 日常日程配置用例 (Schedulers)
在工坊内，玩家需要消耗每日的 AP（行动点数）去维护魔偶。
- **【执行核心机能维护】（花钱消灾）：**
  - 消耗：`1 AP`，`50 金币`
  - 参数影响：`+30 SAN`，`好感度不变`。
- **【进行阳台夜谈】（灵魂交流）：**
  - 消耗：`2 AP`，必须交付一件深渊里找到的非战斗旧物（如：陈旧音乐盒）。
  - 参数影响：`好感度 +50`，大幅度冲长线数值。

### 2. 经典特质样例库 (Traits)
- **正面特质示例 - 【极致洁癖】：**
  - *获得条件：* 连续 3 局深渊探索未损失任何 HP 触发变异。
  - *机制：* 监听局内战斗网络。任何时候只要背包网格内存在**任意一整行/一整列的空格**（未被任何物品沾染），魔偶的全体常驻闪避率提高 20%。
- **负面特质示例 - 【念旧/囤积癖】：**
  - *获得条件：* 连续带同一个 1x1 物品进入了超过 5 次深渊。
  - *机制：* 一旦该物品被移出大网格并试图卖掉，强制触发 `Event_Sale_Reject`，无法卖出。除非在商店花费高额心理疏导费清除这个特质。这导致这个东西永远恶心你占你一个格子。

## 第四部分：异常与边界情况 (Edge Cases)

| 极端情况 (Edge Case) | 系统裁决处理规范 (Resolution Rule) |
| :--- | :--- |
| **玩家疯狂在深渊“虐待”魔偶（常年0界限活动），但靠刷钱在小镇靠送豪礼狂刷好感度？** | **特质分裂 (极端冲突机制)：** 此时系统检测到 `SAN` 数据记录表极低而 `Bond` 极高，判定为“病娇/坏掉的忠诚”。生成专属隐藏特质。其效果是在深渊时每 3 次操作就会强行把鼠标的左右平移反转 5 秒钟——用来惩罚把游戏当做数字计算器玩的功利狂。 |
| **同时获得了两个互相冲突的特质怎么办？**（如特质A要求必须要有空位，特质B要求必须塞满） | **特质优先级/覆盖：** 每个特质具有 `Priority_LV` 属性。当互相覆盖的检查规则生效时，抛弃低层级特质的效果。也可通过同类互斥组解决，同一类别池子里只能挂一张。 |
| **满好感度的奇迹护盾用完了，但还没撤退？** | **无加成的冰冷：** 第二次致死打击完全不讲情面，直接触发生死线结算。而且由于你消耗了这份极其珍贵的信任奇迹依然让魔偶战死，此次战败额外扣除 1整级的 好感度等级（极其严厉的情感制裁）。 |
