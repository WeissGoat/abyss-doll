# 8. 系统实体连接流向与数据架构蓝图

本篇着眼于更高视角的“系统关联”，跳出底层代码层面，清晰梳理庞杂的实体数据在不同模块中究竟如何流通和互相操作。此文档可作为下一步推进具体程序实现开发的理论导向。

## 一、 整体数据关系网络大图谱

游戏由底层向外可以划分三大层的数据领域，它们之间以中央控制机制流转：

### 1. 玩家与大周期承载实体（Global State Record）
- 掌控着全局的系统变量：
  - `当前可用现金资产与欠款状态`
  - `游戏周期时间锚点 (当前的月、日与行动点AP状态)`
  - `小镇全盘物价大盘基数 / 传闻阵营威望库`
  - `仓库整体存储清单 (Stash Vault)` 

### 2. 魔偶中心实体对象 (Doll Object Layer)
这可以说是系统互联运转的“心脏轴心”，所有上下层模块几乎都作用于她。它不是一个静态参数集合，而是一个巨型的动态容器，囊括：
- **物理硬件基座配置库：** 底层定义的背包结构初始空间大网格蓝图，当前装备。
- **心理状态三维坐标系：** 长线常驻不可逆数值（好感度阶层参数）、波动的流状态槽（侵蚀度极值与理智实时数据）、干涉工作与交涉的经营三维值（算力/魅力）。
- **特质外挂事件拦截器列队（Traits Buffer）：** 专门用于监听并强行覆写网格与数值。记录由战斗失败或日常陪伴形成的【恐虫】【小气】等一系列正副标签标识符。

### 3. 局内背包网络层与实例运算 (Grid Battle Engine)
游戏核心机制执行器，负责即时状态演绎发卡与节点推进。
- 在战斗与每一次进入房间，这层引擎便向【魔偶中心对象】进行读取验证。
- 初始化网格区域边界 $\to$ 注入当前在背包装载的物品 $\to$ **运行结算引擎 (Connection Resolver)**：从全局执行“找寻导线、查找同标签数量、触发互相削弱或同态加成的连锁反应事件（Synergy Computation）”。

## 二、 系统联动的全链路流向追踪 (例：一次完整的“贪婪反噬记录”)

以下是一次典型的交互流转全图景演示（展示数据怎么一步步咬合串联的）：

| 发生周期 | 具体触发模块区域 | 关键数据的转移与计算 | 产生的长远影响 |
| :-- | :-- | :-- | :-- |
| **局内深渊层** | 副本掉落与背包整理 | 玩家战胜一个精英，获取物品实例 A[高稀有度/包含毒属性标签]。并由操作员强行放入网格特定坐标系内。 | 当前局内心智扣除变量直接与[毒标签]同步执行每回合削减运算。 |
| **局内撤离判定** | 结算节点区核心判断 | 玩家携带物品A和满额极值 [侵蚀度] 进入撤离阵。 | 副本将物品A归入【公用存储总站库】，并强行向魔偶对象的特质库压入新增的负面事件标签：【特质: 惊恐症生成！】 |
| **日常经营层** | 小镇市场机制贩卖 | 将高价值的物品A放置进小镇店铺，进行资金转化。由于魔偶算力的乘积加上市场浮动变量。获得爆表级大笔金钱回报（从而解除了当月的“时间流转主线破产风险引擎”） | 这部分属于从局内的生存空间压榨流转至了资金安全解脱。 |
| **日常深耕/修缮层** | 工商/抚慰休息处交汇 | 由于魔偶携带新出现的【特质：惊恐症】，她在第二天工作经营的时候各项成功率与客流量骤降20%（数据反向钳制）。 | 玩家必须耗费刚赚到的大幅资金利润用于休养、特殊消弭特质的购买，完成资金水池闭环回抽（防止滚极度膨胀）。 |

## 三、 基于此蓝图的程序预开发建议

当我们通过上述流向确定出没有“幽灵悬空死角体系（即产生数据了但毫无作用）”时，进入实际程序落地应坚决贯彻解耦设计：

1. **ECS 架构引入验证：** 此类复杂的堆叠叠加体系绝不要面向对象死写类继承（Weapon/Potion）。所有进入背包界面的都统称为 `ItemEntity`，通过装配不同的 `ShapeComponent` (形状模块)、`TagComponent` (特定标签词语模块) 和 `OnAdjacentTrigger` (被碰触激发机制模块)，让中心运算机制做扁平化处理遍历是最佳之选。
2. **状态分离准则：** 无论是战斗界面的网格还是魔偶界面面板，仅用来展示【实体对象】后台所处底层矩阵二维数组的推算影像投影而已（展现层只处理旋转拖拽事件与动效，不承载属性核心改表逻辑）。分离是应对成百上千种复杂连锁流派扩展不翻车的硬保障原则。
