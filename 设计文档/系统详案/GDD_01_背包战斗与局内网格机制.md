# 详案_01：背包战斗与局内网格机制 (GDD_01)

> **定位说明：** 本文档定义了游戏最核心的战斗体验——"背包即战场"。涵盖战斗界面布局、战斗节奏、伤害模型、构筑流派体系以及怪物设计框架。本文档与 GDD_06（物品系统）、GDD_08（人偶养成）紧密关联。

---

## 一、战斗界面布局

### 1.1 屏幕分区
屏幕以约 4:6 比例划分为两大区域：

**左侧区——战斗舞台（约40%宽度）**
- 魔偶角色展示区（立绘/骨骼动画），下方显示两个核心状态条：
  - **HP**（血量/护盾）：被攻击时消耗
  - **SAN**（理智值）：受深渊环境和特定怪物技能消耗
- 敌人展示区：当前遭遇的所有怪物阵列，每个怪物头上显示「意图标识」（下一步打算：攻击?施法?防御?召唤?）
- **战利品暂存槽（Inbox）**：位于左下角，战斗胜利后掉落的物品悬浮在此。玩家需要手动拖入背包，离开当前节点时Inbox清空不保留

**右侧区——背包网格（约60%宽度）**
- 当前底盘决定的网格画布（初始4×4或4×5，随底盘更换而变）
- 正常格子（亮色可用）、结晶封锁格（紫黑色无法使用）、高温核心格（特殊底盘独有）等不同状态格子混合
- 网格中已摆放的物品实时显示充能进度条（每件武器独立的圆环/条形进度）

### 1.2 信息表达原则
- 所有战斗信息都必须能在**一屏之内**读完——不用翻页、不用切换Tab
- 怪物的"意图"是最关键的信息，必须在怪物头上用图标+数字明确展示（如：🗡️12 = 即将造成12点物理伤害）
- 背包网格中物品之间的连接关系（光环/链路），用发光连线或色彩变化实时反映

## 二、战斗节奏设计

### 2.1 半即时推进制（ATB式）
战斗**不是回合制**，也**不是纯即时**，而是介于两者之间的「半即时推进」：

- 时间持续流动，每件武器按照自己的充能速度独立推进进度条
- 充能完毕后**自动开火/自动防御**，玩家无需手动点击攻击
- 玩家的核心操作不是"按攻击键"，而是**调整摆放、使用消耗品、管理空间**

### 2.2 子弹时间机制
- 玩家**按住鼠标拖拽**背包中任意物品时，战斗时间流速降至原来的5%（极度减速但不完全暂停）
- 子弹时间内，玩家可以：旋转物品、调整摆放位置、将Inbox中的战利品塞入背包
- 为什么不完全暂停？保留极微弱的时间流动，维持"随时可能被打"的紧张感，防止玩家无限思考

### 2.3 消耗品的手动操作
- 武器和防具全部自动运作，但**消耗品必须手动使用**
- 使用方式：将消耗品拖拽到目标上（拖到魔偶=自用、拖到怪物=投掷攻击）
- 使用后消耗品从背包消失，**立刻释放所占格子空间**——这是"空间释放"机制的核心，鼓励玩家在搜刮前主动消耗药水腾位

### 2.4 战斗中的摆放调整
- 战斗进行中，玩家可以在拖拽降速期间重新排列背包内的物品
- **代价**：被拿起的物品在悬空状态时，其充能进度冻结、光环效果临时失效
- 这意味着"战斗中调阵"是有风险的——调阵期间你的火力和防御都暂时减弱

## 三、伤害与生存模型

### 3.1 伤害类型
| 类型 | 来源 | 被什么抵挡 |
|---|---|---|
| 物理伤害 | 近战怪物撞击、投射物 | 护盾护甲（Shield） |
| 魔法伤害 | 法术攻击 | 直接扣HP（护盾无效），但可被特定义体减免 |
| 侵蚀伤害 | 环境瘴气、恐怖类怪物 | 直接扣SAN（无法用护盾挡） |
| 真实伤害 | 极少数Boss技能 | 无视一切防御直接扣HP |

### 3.2 魔偶的生存资源
| 资源 | 功能 | 归零后果 |
|---|---|---|
| **HP** | 生命值，被攻击时消耗 | HP归零 → 战斗失败，触发死亡惩罚（详见GDD_02） |
| **护盾（Shield）** | 抵挡物理伤害的缓冲层，先消耗护盾再扣HP | 护盾归零 → 物理伤害直接打在HP上 |
| **SAN** | 理智值，受深渊环境和特定攻击消耗 | SAN归零 → 触发理智崩溃事件（详见GDD_03） |
| **耐力（Stamina）** | 部分重型武器开火需要消耗 | 耐力不足 → 对应武器充满能量也无法开火，卡死 |

### 3.3 耐力系统
- 耐力是魔偶自身的能量池，初始值由底盘和义体决定
- 重型武器每次开火消耗耐力（轻型武器不消耗）
- 耐力每秒自动缓慢恢复
- **设计意图**：耐力限制了"全部带重武器"的暴力堆叠，迫使玩家在重火力和可持续性之间做平衡

### 3.4 护盾再生
- 护盾不会自动恢复
- 护盾只能通过特定物品效果（如「护盾电池」消耗品、「护盾发生器」被动装备）来恢复
- **设计意图**：护盾是一种"带了才有，不带就裸挨打"的防御资源，鼓励玩家在背包中预留防御物品的位置

## 四、构筑（Build）流派体系

构筑是玩家通过物品的选择和摆放方式形成的战斗风格。不同的流派对背包空间的利用方式截然不同。

### 4.1 三大核心流派

#### 流派A：链路管道流（Circuit/Flow Build）
- **核心原理**：物品之间存在方向性连接关系，形成一条"生产线"
- **典型构筑**：`[魔力核]` → `[能量导管]` → `[重型魔导炮]`。核心产出能量，导管传输，末端武器消耗能量开火
- **优势**：伤害上限极高，单次开火的爆发力无人能敌
- **代价**：整条链路占用大量空间且位置关系死板；如果为了装战利品拆掉一截导管，整套火力系统直接瘫痪
- **适合的搜打撤策略**：少打精英多撤离，每次只拿小型高价值战利品

#### 流派B：相邻光环流（Adjacency/Aura Build）
- **核心原理**：物品通过"相邻"关系互相增益，不要求方向性和顺序
- **典型构筑**：一块`[淬毒磨刀石]`放中间，四周围一圈`[小匕首]`。磨刀石给所有相邻武器附毒，匕首们快速平刺
- **优势**：形状极其灵活，物品可以见缝插针，对搜刮空间的兼容性最好
- **代价**：单体伤害不高，打精英/Boss比较吃力
- **适合的搜打撤策略**：频繁下潜清小怪，靠量取胜

#### 流派C：资源转化流（Resource Conversion Build）
- **核心原理**：不依赖摆放位置，而是围绕全局数值（如护甲值、热量、侵蚀度）做文章
- **典型构筑**：大量积攒"热量"的护盾 + 一把被动为"总热量越高暴击率越高"的双手大剑
- **优势**：不怕被怪物打乱背包布局，全局数值不因位移而改变
- **代价**：需要特定的组合才能成型，随机性依赖度高
- **适合的搜打撤策略**：大胆深入，利用数值滚雪球

### 4.2 三大特色构筑（与搜打撤独特融合）

#### 垃圾发射器流（Trash Cannon Build）
- **核心装备**：`[黑洞粉碎炮]`（3×2），本身无伤害。每回合自动摧毁背包内最便宜的一件物品，将其转化为高额伤害弹
- **体验**：下深渊时包里空空如也，到了深渊疯狂捡各种垃圾当子弹打。完美融合搜刮与战斗

#### 虚空留白流（Negative Space Build）
- **核心装备**：`[孤高之刃]`（1×3）等"空越多越强"的武器
- **体验**：背包中空余格子越多，战斗力越爆表。但一旦捡了战利品填补了空位，战斗力断崖下跌。撤离路变成纯心跳玩法

#### 情感羁绊流（Empathy Build）
- **核心装备**：好感度赠礼（如八音盒2×2，无战斗属性）
- **体验**：必须配合局外高好感度。魔偶处于丝血/高侵蚀时，情感道具触发"羁绊共鸣"，周围所有物品临时升级。故意走钢丝来触发"爆种"清图

### 4.3 构筑的破坏与重建
背包战斗最独特的体验在于：**你的构筑会在一局之中被不断破坏和重建**。

破坏来源：
- 怪物技能封锁/移动格子中的物品
- 为了装战利品主动拆掉构筑的一部分
- 诅咒物品的辐射迫使你重新调整布局
- 消耗品用完后产生的空位改变了相邻关系

这种**"动态构筑"**——而非传统RPG那种"配好装就不动了"的固态构筑——是本游戏战斗乐趣的核心所在。

## 五、怪物设计框架

### 5.1 怪物与背包的互动原则
本游戏的怪物不仅是"血条 + 攻击力"。好的怪物设计必须**干涉玩家的背包空间**，让战斗和背包管理真正融为一体。

怪物的行为分为两大类：
- **对人攻击**：削减魔偶的HP/护盾/SAN/耐力（传统伤害）
- **对包干涉**：破坏、占据、污染、移动背包中的格子或物品（独特设计）

### 5.2 怪物行为类型图鉴

#### 物理攻击型
- 直接对魔偶造成物理伤害
- 意图标识清晰（🗡️+数字）
- 代表怪：深渊狂犬、石甲守卫

#### 法术攻击型
- 造成无视护盾的魔法伤害
- 通常有较长的蓄力时间作为警告
- 代表怪：瘴气法师、念动水母

#### 格子侵占型（核心设计）
- 向背包中注入无法使用的"垃圾占位物"
- 例如：史莱姆每隔8秒向背包中空位喷射"毒液凝胶"（1×1）占据空格。玩家必须拖出丢弃，但丢弃时扣SAN
- **设计意图**：直接攻击背包空间这一核心资源

#### 格子封锁型
- 将背包中已有的格子临时变为"深渊结晶"状态，无法使用
- 已经放在被封锁格子上的物品被强制弹出到Inbox
- 代表怪：结晶蜈蚣（每20秒封锁边缘1个随机格子）
- **设计意图**：直接破坏玩家的构筑布局，迫使在战斗中紧急重组

#### 物品移动型
- 随机交换背包中两件物品的位置
- 或者将某件物品强制推移1~2格
- 代表怪：混沌精灵（每15秒随机交换两件物品位置）
- **设计意图**：直接摧毁依赖精确摆放的链路流构筑

#### 侵蚀型
- 不造成HP伤害，但持续大量消耗SAN值
- 通常体型小、血量低，但数量多
- 代表怪：窃魂虫群（每秒SAN-1，但HP只有3）
- **设计意图**：逼玩家快速清理，时间就是SAN

#### Boss型
- 融合上述多种行为，并拥有阶段切换机制
- 每个Boss至少有一个独特的"对包干涉"招式，迫使玩家针对性准备
- Boss击杀后必定掉落稀有战利品（通常形状极大极奇特）

### 5.3 怪物的意图系统
- 每个怪物在行动前会**提前展示下一步行动意图**（参考《杀戮尖塔》）
- 意图以图标+数字的形式显示在怪物头上
- 意图分类：
  - 🗡️ 攻击（附数字=伤害量）
  - 🛡️ 防御（怪物给自己加护甲）
  - 💀 特殊技能（对包干涉类，用独特图标标识具体类型）
  - ❓ 未知（部分怪物的初始行动不预告，制造不确定性）

### 5.4 怪物分层设计
| 层级 | 怪物类型 | 行为特征 | 设计目的 |
|---|---|---|---|
| 杂兵（Minor） | 深渊1-3层的基础怪物 | 只有物理攻击和简单的格子侵占 | 教学向，让玩家熟悉战斗节奏 |
| 精英（Elite） | 深渊3-6层出现 | 拥有2种以上行为，更高频次的对包干涉 | 真正的策略考验，高风险高回报 |
| Boss | 每层关底 | 阶段切换 + 全类型行为组合 | 决定玩家是否有资格进入下一层 |
| 隐藏Boss | 特殊条件触发 | 极端强度，但掉落传说级物品 | 高风险极端博弈 |

## 六、战斗胜负条件与结算

### 6.1 胜利条件
- 当前节点所有怪物HP归零 → 战斗胜利
- 战利品在Inbox中生成，玩家进入搜刮阶段

### 6.2 失败条件
- 魔偶HP归零 → 战斗失败，触发撤离失败惩罚（详见GDD_02）
- 魔偶SAN归零 → 触发理智崩溃事件链（详见GDD_03）

### 6.3 战后搜刮阶段
战斗胜利后，界面不立刻切换。进入「搜刮阶段」：
1. 战利品在左下角Inbox中展示（显示形状、标签、预估价值）
2. 时间暂停，玩家可以自由拖拽、旋转、排列
3. 玩家面临核心抉择：
   - 背包满了 → 是否扔掉现有武器来装高价值战利品？
   - 战利品形状奇怪 → 是否值得为了装它重组整个构筑布局？
   - 部分战利品带有`[诅咒]`标签 → 是否愿意承受辐射效果来带走它？
4. 关闭搜刮界面 → Inbox中未装入背包的物品**永久销毁**

## 七、战斗节奏的控制

### 7.1 一场战斗的时长预期
| 怪物类型 | 预期时长 | 说明 |
|---|---|---|
| 杂兵战 | 15~30秒 | 快节奏，不应让玩家感到疲劳 |
| 精英战 | 45~90秒 | 有一定策略深度，需要主动使用消耗品 |
| Boss战 | 2~4分钟 | 允许有阶段切换和紧急调阵 |

### 7.2 为什么选择半自动而非手动操作
如果每回合让玩家手动选择：用哪把武器 → 攻击哪个怪物 → 确认。一场普通杂兵战就要2~3分钟。在搜打撤框架下（一局可能打10~15场战斗），这会导致严重的操作疲劳。

半自动系统的优势：
- 杂兵战近乎自动完成，玩家只需偶尔喝药
- 精英战和Boss战中，玩家的操作集中在"消耗品使用"和"紧急调阵"这两件高价值的事上
- 让"摆放构筑"成为局前/搜刮阶段的核心乐趣，而非战斗中的负担

## 八、边界规则

| 情况 | 规则 |
|---|---|
| 物品被拖起悬空时，怪物攻击了怎么办？ | 悬空物品的充能冻结、光环失效。但物品本身不会被怪物的"物品移动"技能影响（手里的东西不受位移干扰） |
| 怪物刚好封锁了我正在往下放的格子 | 放置被驳回，物品退回鼠标位置，发出红色警告反馈 |
| 战利品形状和我背包剩余空间完全不匹配 | 这是设计的一部分——不是所有战利品都"装得下"。玩家需要做断舍离 |
| 背包内所有武器都因为耐力不足而卡死了 | 连续5秒所有武器处于满充能但无法开火的状态，最高充能的武器自动过载爆炸（对魔偶造成少量HP伤害），释放一部分耐力 |
| 战斗中把所有武器都扔了只剩战利品 | 魔偶没有攻击手段，只能靠护盾/HP硬抗等待——但没有输出最终必死。这是自作自受的决策后果 |
