# 系统关联总图：全系统内在关联与数据流向 (GDD_00)

> **定位说明：** 本文档是全部12份详案的"上帝视角"。它不重复各系统的内部设计，而是专注于描述**系统之间的连接关系**——什么流向什么、谁驱动谁、共享哪些资源。开发时用于理解"改动一个系统时会波及哪些系统"；设计时用于验证"是否存在断裂的循环或失控的正反馈"。

## 一、核心循环总览

游戏的宏观循环由三个引擎驱动，它们彼此对立又彼此依赖：

```
              ┌──────────────────────────────────┐
              │         时间日程 (GDD_07)          │
              │     每天3AP，每月交租，12个月结局     │
              └──────┬───────┬───────┬────────────┘
                     │       │       │
            ┌────────▼──┐ ┌─▼──────┐ ┌▼──────────┐
            │ 深渊探索    │ │ 小镇经营 │ │ 养成与交互  │
            │ (GDD_01/02)│ │(GDD_04)│ │(GDD_03/08 │
            │            │ │        │ │ /11/12)   │
            │ 消耗：      │ │ 消耗：  │ │ 消耗：     │
            │ 2AP+状态    │ │ 1AP    │ │ 1AP+金币   │
            │            │ │        │ │ +素材      │
            │ 产出：      │ │ 产出：  │ │ 产出：     │
            │ 战利品+碎片 │ │ 金币   │ │ 好感度+状态 │
            └─────┬──────┘ └──┬─────┘ └──┬────────┘
                  │           │          │
                  └─────┬─────┘          │
                        │                │
                        ▼                │
              ┌───────────────┐          │
              │  物品系统       │          │
              │  (GDD_06)     │◄─────────┘
              │  万物的数据基础  │  （素材消耗/赠礼）
              └───────────────┘
```

**三足鼎立的设计意图**：
- 深渊产出战利品和素材 → 但消耗状态（HP/SAN/磨损）
- 经营将战利品变现为金币 → 但消耗时间（AP）
- 养成用金币和素材修复/强化 → 使下次深渊更安全
- 三者共享一个不可扩张的资源：**每天3个行动点**

## 三、六大共享资源——系统博弈的根源

所有系统的矛盾都源于**对同一池资源的争夺**。以下是全游戏最核心的六种共享资源：

### 资源①：行动点（AP）
| 消耗方 | 每次消耗 | 竞争关系 |
|---|---|---|
| 深渊探索 | 2AP | 占用一天的绝大部分 |
| 小镇经营 | 1AP | 与深渊抢上午/下午 |
| 养成（保养/特训） | 1AP | 与经营/交互抢时间 |
| 交互（对话/陪伴） | 1AP | 与保养抢夜晚时段 |
| 小镇外出 | 1AP | 与所有活动竞争 |

**冲突核心**：一天只有3AP，做了A就做不了B。

---

### 资源②：背包网格空间
| 竞争方 | 占据方式 | 矛盾 |
|---|---|---|
| 武器/防具 | 形状各异的持久占据 | 战斗力 vs 搜刮空间 |
| 消耗品 | 使用后释放空间 | 安全 vs 灵活 |
| 情感锚点物 | 无战斗力的持久占据 | 好感度 vs 效率 |
| 委托物 | 大型固定占据 | 势力奖励 vs 生存风险 |
| 战利品 | 战斗后的竞争占据 | 每件战利品都在和武器抢位 |
| 怪物干涉（毒液/结晶） | 强制占据 | 被动损失空间 |

**冲突核心**：格子是零和博弈。每多一格武器，就少一格搜刮空间。

---

### 资源③：金币
| 收入来源 | 支出去向 | 占比（中期预估） |
|---|---|---|
| 商店出售战利品 | 月租金 | ~30% |
| 势力委托奖励 | 机体维护（保养费） | ~25% |
| 黑市交易 | 义体/底盘制造 | ~20% |
| — | 消耗品采购 | ~15% |
| — | 换装/家具（奢侈品） | ~10% |

**冲突核心**：收入全靠深渊和经营，支出有强制项（租金+维护）和可选项（养成+奢侈）。强制项确保金币永远不够花。

---

### 资源④：SAN值（理智）
| 消耗来源 | 恢复来源 |
|---|---|
| 深渊过路税（每节点-2~4） | 休息行动（+15~40，受床铺影响） |
| 战斗中恐怖类怪物攻击 | 保养·净化（降低侵蚀度） |
| 背包中诅咒物品辐射 | 安全区通讯（好感度高时+3~5） |
| 丢弃毒液凝胶物（每个-1） | 情感陪伴/噩梦安抚 |
| 高侵蚀度状态的被动流失 | 心智天赋树·深渊抗性分支 |

**冲突核心**：深渊是SAN的主要消耗场景，恢复SAN需要地面的AP和金币。SAN是"深渊可持续性"的计时器。

---

### 资源⑤：深渊素材
| 产出 | 消耗 |
|---|---|
| 深渊战斗掉落 | 义体插件制造 |
| 深渊宝箱 | 底盘升级 |
| 特殊节点事件 | 机体保养（净化结晶等） |
| — | 势力委托交付 |

**冲突核心**：同一块素材，交给势力换声望、留在工坊造义体、还是紧急用来做净化材料？三方争夺同一个池。

---

### 资源⑥：记忆碎片
| 产出 | 消耗 |
|---|---|
| 深渊探索（无论成败自动获取） | 心智天赋树节点解锁 |
| 成功撤离（额外奖励） | — |

**特殊性**：记忆碎片是唯一**不被任何其他系统争夺**的纯净资源。它只有一个来源（深渊），只有一个去向（天赋树）。这是故意的——它是失败者的慰藉，不应被经济系统或其他模块侵蚀。

## 四、系统间的关键数据流

### 4.1 深渊 → 地面 数据流（下潜结束时）

```
深渊结束
  │
  ├─ 成功撤离 ────────────────────────────────────────┐
  │   ├─ 背包物品 ──→ 地面仓库                          │
  │   ├─ 安全盒物品 ──→ 地面仓库                        │
  │   ├─ 记忆碎片 ──→ 天赋树可用余额                     │
  │   ├─ 魔偶状态（HP/SAN/磨损/侵蚀）──→ 带回地面等待修复  │
  │   └─ 特质触发检测 ──→ 是否新增/转化特质               │
  │                                                    │
  └─ 战斗失败 ──────────────────────────────────────────┤
      ├─ 新获战利品 ──→ 全部销毁                         │
      ├─ 绑定装备 ──→ 保留但附加损坏标签                  │
      ├─ 安全盒物品 ──→ 保留                             │
      ├─ 情感锚点物 ──→ 保留（自动绑定）                  │
      ├─ 记忆碎片 ──→ 保留                              │
      ├─ 魔偶状态 ──→ HP清零 + SAN暴跌 + 磨损暴增        │
      └─ 惩罚账单 ──→ 维修费 + 好感度下降                 │
```

### 4.2 地面日常循环数据流（每天）

```
每日开始
  │
  ├─ 日程分配（3AP）──→ 各系统获得执行许可
  │
  ├─ 若选择[下深渊] ──→ 整装待发 → 深渊探索循环
  │
  ├─ 若选择[开店] ──→ 经营系统启动
  │   └─ 物品从仓库上架 → 顾客AI判定 → 金币入账
  │
  ├─ 若选择[保养] ──→ 交互系统·保养流程
  │   └─ 材料消耗 → 磨损/侵蚀降低 → 好感度微调
  │
  ├─ 若选择[陪伴] ──→ 交互系统·对话流程
  │   └─ 对话池抽取 → 好感度变化 → 可能触发剧情
  │
  ├─ 若选择[外出] ──→ 势力拜访/商店购买/情报收集
  │   └─ 声望变化 → 委托接取 → 商品购买
  │
  └─ 每日结束 → 强制事件检查（债主/崩溃/特殊来访）
```

### 4.3 周结算数据流

```
周末结算触发
  │
  ├─ 物价刷新 ──→ 经营系统（商品价格变化）
  ├─ 委托刷新 ──→ 势力系统（新订单上架）
  ├─ 传闻发布 ──→ 剧情系统（情报板更新）
  ├─ 过劳判定 ──→ 日程系统（是否触发强制休息）
  ├─ 特质累积检测 ──→ 标签特质系统（是否达到阈值）
  └─ 状态累积落地 ──→ 养成系统（侵蚀/磨损写入）
```

### 4.4 月结算数据流

```
月末结算触发
  │
  ├─ 租金扣除 ──→ 金币减少（若不足→违约处理）
  ├─ 阶位进化判定 ──→ 养成系统（底盘/属性是否达标）
  ├─ 结局条件推进 ──→ 剧情系统（检查终局分支）
  └─ 月度成就记录 ──→ 存档数据（最深层数/总收入等）
```

## 五、三大闭环——验证循环健康度

一个健康的游戏经济需要**每个资源都有明确的产出口和消耗口**，形成闭环。以下是本游戏最关键的三条大循环：

### 闭环A：战斗力循环
```
下深渊 → 获得素材/核心 → 制造义体/升级底盘 → 战斗力提升 → 能打更深层 → 更好的素材
```
**健康检查**：如果义体/底盘的制造成本过低，玩家会过早毕业；如果过高，中期会感到绝望。

### 闭环B：经济循环
```
下深渊 → 获得战利品 → 商店出售 → 金币 → 交租金+买消耗品+保养 → 维持下潜能力 → 再下深渊
```
**健康检查**：如果深渊的净收益（卖出-维护费-租金）长期为正且递增，经济会崩盘（后期无压力）。租金递增和维护费递增是确保这条循环永远"刚好够用但不富裕"的关键阀门。

### 闭环C：情感循环
```
下深渊（伤害魔偶）→ 回到地面保养/陪伴 → 好感度提升 → 解锁奇迹机制 → 下次深渊更安全 → 但深渊又会消耗SAN → 又需要保养
```
**健康检查**：如果好感度无衰减且投入极低，"奇迹机制"会变成白送的保险。保养的金币消耗和AP消耗确保情感也有"维护成本"。

## 七、潜在的危险联动——需要警惕的设计陷阱

| 危险场景 | 涉及系统 | 失控表现 | 防范手段 |
|---|---|---|---|
| **金币通胀** | 04经营 + 07日程 | 中后期金币积累远超支出，租金和维护变得无关痛痒 | 租金递增 + 维护费随深渊层数递增 + 换装/家具作为金币回收池 |
| **好感度速通** | 12交互 + 03好感 | 砸钱暴刷好感度，过早解锁奇迹/真结局 | 赠礼边际递减 + 心事对话为一次性 + 好感突破需事件触发而非纯数值 |
| **深渊无压力** | 08养成 + 01战斗 | 底盘+义体满配后深渊变成无脑推图 | 深层怪物的"对包干涉"能力与玩家养成同步升级；天赋树不提供暴力数值加成 |
| **时间悠闲** | 07日程 | 玩家找到了一套"完美日程"每天重复 | 过劳机制 + 限时事件 + 委托截止日打断节奏 |
| **特质滥用** | 09特质 | 故意刷负面特质再转化为正面（如反复被虫杀→虫克） | 战败惩罚（磨损+维修费）确保"刷特质"的净成本为负 |
| **势力全满** | 10势力 | 全势力满声望，所有图纸/商品解锁 | 声望总量硬控（一局只够2~3个满） |

## 九、数值设计哲学——等价交换本位与经济通缩模型

> 本节从底层数学角度阐述游戏经济的设计原则。在剥离所有包装后，游戏的基底等价物本质**并非"金币"**，而是两样更根本的要素。

### 9.1 等价交换本位模型

游戏所有系统的博弈，最终都可以归结为两种基础介质的汇率竞争：**背包网格数 (Space)** 与 **行动点数 (AP / Time)**。

- **空间本位换算**：`1个 1x1 格子` = 当局内承伤缓冲上限X = 牺牲DPS输出转化率Y = 折合局外金币转化比例Z。如果将一个 2x2 的小熊玩偶放进背包，等于用极其昂贵的战力空当（4格空间）兑换了强大的精神防御屏障与局外养成红利。
- **时间本位换算**：
  - 消耗 2AP 下深渊 = 极高经济入账期望 + 极高精神损伤惩罚
  - 消耗 1AP 小镇经商 = 中低额安全盈利 + 轻微磨损
  - 消耗 1AP 工坊陪伴治愈 = 纯粹的输出端（花钱并消耗行动），回退 SAN 值并长期累积隐藏好感阈值

**核心原则**：任何新增系统或物品的价值估算，都应先折算为"它等价于多少格背包空间"或"它等价于多少AP"，再判断是否合理。

### 9.2 蓄水池模型（经济通缩抗膨胀）

为防止长期游戏后的货币冗余导致下深渊不再刺激，必须建立动态的"抽水机"：

- **产出端（水龙头）**：相对唯一——深渊中获取的实体物品经商店交易转化为金币。
- **消耗端（下水口）**：分为三级阶梯——
  - `小型刚需滴流`：魔偶修理、血药消耗品、保养费。不买就死（或严重掉战斗力）。
  - `巨型强制泵浦`：月末高额账单、税收、通行证费。随着难度指数级跃升（不够就 Game Over）。
  - `无底洞情感沉没`：买好看但战斗力为零的首饰和服装送魔偶，是防止囤金的终极情感利器。

**健康标准**：在任何游戏阶段，玩家的净收入（深渊产出 - 维护费 - 租金）都应该处于"刚好够用但不富裕"的微妙平衡点。

### 9.3 反滚雪球对抗函数

最关键的设计在于不让玩家实现长期的"以战促战"稳定滚大循环：

- 利用 **理智/侵蚀度引擎** 作为反向乘数：玩家越贪婪（单次副本走的房间越长、背包塞满高价有毒物品），SAN 值受到的乘数削弱惩罚越高。
- 当 SAN 值到达红线，反而会加速局内部件失效或强制废弃网格的速度，造成输出/护盾断档。
- 因此，金钱的堆积速度和探索的安全保障存在**强制负相关曲线**，迫使任何阶段的玩家都会经常遭遇红血心跳的抉择体验。

## 十、附录：全链路数据流追踪案例——"一次完整的贪婪反噬记录"

> 以下案例以叙事化方式展示数据在四个阶段中如何逐步咬合串联。适合用于策划对齐设计共识。

| 发生周期 | 触发模块 | 关键数据的转移与计算 | 产生的长远影响 |
|:--|:--|:--|:--|
| **局内深渊层** | 副本掉落与背包整理 | 玩家战胜一个精英，获取物品A（高稀有度/包含毒属性标签）。强行放入网格特定坐标。 | 当前局内心智扣除变量直接与毒标签同步，执行每回合削减运算。 |
| **局内撤离判定** | 结算节点核心判断 | 玩家携带物品A和极值侵蚀度进入安全区撤离。 | 副本将物品A归入公用存储总站库，并强行向魔偶特质库压入新增负面标签：**【特质: 惊恐症生成！】** |
| **日常经营层** | 小镇市场贩卖 | 将高价值物品A放入店铺，进行资金转化。由于魔偶算力乘积加上市场浮动变量，获得大笔金钱回报。 | 成功解除当月"时间流转主线破产风险引擎"。从局内的生存空间压榨流转至资金安全。 |
| **日常修缮层** | 工坊抚慰交汇 | 由于魔偶携带新出现的【特质：惊恐症】，第二天工作经营各项成功率与客流量骤降20%。 | 玩家必须耗费刚赚到的利润用于休养、购买消弭特质的药品，完成资金水池闭环回抽。**防止经济滚雪球膨胀。** |

**案例总结**：一颗深渊里捡到的"毒宝石"，从局内的空间压榨→撤离时的心智代价→经营中的高价变现→修缮时的利润回吐，完美走完了四个系统的数据流转闭环。没有一个环节的数据是"悬空"的。
