# 详案_12：人偶交互管理器 (GDD_12)

> **定位说明：** 本文档定义玩家与魔偶之间所有"直接接触"行为的设计规范——包括触摸、对话、赠礼、保养等工坊场景中的互动。这些交互是"好感度"和"人偶状态"两个核心数值的主要操作入口，也是玩家与魔偶之间**情感连接**的最直接体现。
>
> **与其他文档的关系：**
> - 交互产出的好感度/SAN → 写入 GDD_03（人偶实体与好感双轨）
> - 交互消耗的AP → 受 GDD_07（时间日程）约束
> - 保养交互的材料消耗 → 读取 GDD_08（养成·机体维护）
> - 赠礼消耗的物品 → 读取 GDD_06（物品系统）
> - 交互触发的对话 → 由 GDD_05（剧本调度）驱动

---

## 一、交互发生的场景

### 1.1 工坊主界面——交互的主战场
绝大多数交互发生在「工坊主界面」——日常的非战斗、非经营状态下，玩家面对魔偶的那块屏幕。

工坊界面中魔偶居中站立（或坐着），占据屏幕主体。玩家的鼠标/手指就是与她交互的唯一工具。

### 1.2 其他场景中的有限交互
| 场景 | 可用交互 | 不可用交互 |
|---|---|---|
| 深渊安全区 | 简短对话（通讯器） | 触摸、赠礼、保养 |
| 商店经营中 | 无（她在忙着招待客人） | 全部 |
| 人偶房间 | 点击触发反馈、观察待机 | 赠礼、保养（需在工坊） |
| 整装待发界面 | 简短情感反馈（自动） | 主动交互 |

---

## 二、交互行为分类

所有交互分为 **五大类**，按操作方式和产出效果区分：

---

### 2.1 触摸交互（Touch）

#### 定位
最轻量、最即时的交互方式。不消耗AP，不消耗资源，随时可做。是玩家与魔偶之间最日常的"存在确认"。

#### 操作方式
- 鼠标点击/悬停在魔偶的不同身体区域

#### 触摸区域与反馈

| 触摸区域 | 正常状态反馈 | SAN偏低时的反馈 | 好感度极低时的反馈 |
|---|---|---|---|
| **头顶** | 微微低头，小声说"嗯……"（害羞） | 缩了一下，但没有躲开 | 偏头躲避，沉默 |
| **脸颊** | 脸红，"主上……干嘛啦" | 呆滞无反应，眼神涣散 | 拨开手，"别碰我" |
| **肩膀** | 点头，"有什么吩咐吗？" | 轻微颤抖 | 冷淡回应 |
| **手** | 握住你的手，安静片刻 | 手是冷的，她低声说"好冷……" | 甩开 |
| **背部** | "谢谢……这样很舒服" | 僵硬但没有拒绝 | 转身背对你 |

#### 数值影响
- 单次触摸：好感度 +1~3（极微量，且有每日上限——每天前5次有效，之后不再增长）
- 过度触摸惩罚：同一区域连续点击超过3次 → 魔偶表现出不耐烦，好感度不再变化
- SAN极低时触摸脸颊/头顶 → 可能触发应激反应对话（"不要……不要靠近我……"），扣2点好感度

#### 设计意图
触摸是"不花任何成本的微小善意"。它的数值贡献几乎可以忽略，但它的**情感表达功能**不可替代。当玩家在深渊大败而归、一无所有的时候，至少还可以摸摸她的头。

---

### 2.2 对话交互（Talk）

#### 定位
消耗1个AP的正式交流行为，是推进好感度和解锁剧情的主要手段。

#### 对话的触发方式
- 在日程中安排「情感陪伴」行动（消耗1AP）
- 进入工坊后点击「对话」按钮

#### 对话池机制
对话不是固定脚本，而是从「当前可用对话池」中根据条件抽取：

| 对话类型 | 触发条件 | 好感度产出 | 频率 |
|---|---|---|---|
| **日常闲聊** | 无特殊条件 | +5~10 | 最常见，多次重复后台词会变化 |
| **深渊回忆** | 刚从深渊回来（当天或前一天） | +10~15 | 根据本次深渊经历动态生成话题 |
| **经营感想** | 当天开过店 | +5~8 | 谈论客人趣事或卖出的战利品 |
| **心事吐露** | 好感度达到特定等级的首次 | +20~30（一次性） | 里程碑对话，附带CG或特殊演出 |
| **争执/冷战** | 好感度极低或连续忽视 | 0（但不扣） | 魔偶主动表达不满 |
| **噩梦安抚** | SAN值极低 + 夜晚AP③ | +15 + SAN恢复10 | 她做了噩梦，需要你的陪伴 |

#### 夜晚对话加成
在夜晚（AP③）进行的对话，好感度产出**翻倍**。世界观包装为："夜深人静时的交心，比白天的寒暄更有分量。"

#### 对话中的选项
部分对话会出现2~3个选项，但本游戏的对话选项**不改变剧情走向**（参见GDD_05的"宽线性"设计）。选项影响的是：
- 本次对话的好感度增量（+5 或 +15 的差别）
- 对话的情感色彩（温柔/务实/逗趣）
- 极少数选项会影响特质的触发概率

---

### 2.3 赠礼交互（Gift）

#### 定位
用物质表达情感——把深渊里辛苦搜刮回来的东西送给她。这是"金钱/资源 → 好感度"的直接转化管道。

#### 操作方式
- 打开仓库面板，将物品拖拽到魔偶身上释放
- 或在赠礼界面中选择物品点击赠送

#### 赠礼效果计算
好感度增量由**物品属性 × 魔偶偏好 × 当前状态**三者共同决定：

##### A. 物品自身的情感价值
| 物品类型 | 基础好感增量 | 说明 |
|---|---|---|
| 情感锚点物（`[Bond]`标签） | +30~50 | 专门设计用于赠礼的高价值情感道具 |
| 深渊战利品（非贸易品） | +10~20 | "这是我在深渊里找到的，想给你看看" |
| 贸易品（金币价值高） | +5~10 | "值钱但不走心"，她能感受到区别 |
| 消耗品（药水等） | +3~5 | "实用但没有惊喜" |
| 诅咒物品 | -10~-20 | "你为什么要把这种东西塞给我？！" |

##### B. 魔偶的个人偏好
每个魔偶角色（在多周目选择不同魔偶时）拥有**隐藏的偏好标签**：
- 偏好`[机械]`的魔偶 → 收到精密零件时额外加成
- 偏好`[血肉]`（好奇类性格）→ 收到怪物标本时反而很开心
- 讨厌`[虫类相关]`→ 收到蜘蛛腿会直接扣好感

偏好在初期是隐藏的，玩家通过多次赠礼的反馈试探，或通过对话中的暗示逐渐发现。

##### C. 当前状态修正
| 状态 | 修正 | 原因 |
|---|---|---|
| 好感度Lv.1~3 | ×0.8 | 关系不深时，礼物的感动力有限 |
| 好感度Lv.4~7 | ×1.0 | 正常 |
| 好感度Lv.8~10 | ×1.3 | 深度信赖时，同样的东西更让她感动 |
| 拥有「心墙」特质 | ×0.5 | 信任崩塌后重建极难 |
| 当天已送过礼 | 第2件起 ×0.3 | 防止"砸钱暴刷"——一天多次送礼边际效益急剧递减 |

#### 赠礼的表现反馈
- **喜欢的礼物**：双手接过，抱在胸前，眼睛发亮，播放专属开心语音
- **一般的礼物**：单手接过，礼貌地说"谢谢"
- **不喜欢的礼物**：犹豫地接过，表情微妙，"这个……嗯，谢谢。"
- **讨厌的礼物**：直接推回来，"主上，请不要把这种东西给我。"

#### 特殊赠礼场景
- **送出第一把使用过的旧武器**：触发专属对话——"这是……主上用过的武器？我会好好保管的。"（房间展柜自动多一件纪念物）
- **送出深渊遗物**：触发法师塔相关对话线索
- **送花**：房间窗台出现干花纪念物（详见GDD_11）

### 2.4 保养交互（Maintenance）

#### 定位
保养不是冷冰冰的"点按钮扣钱"，它是一整套有温度的护理流程。玩家通过保养来修复魔偶的物理损伤和精神侵蚀，同时在操作过程中与她产生情感互动。

#### 操作方式
在日程中安排「机体维护」行动（消耗1AP）后，进入保养交互界面。

#### 保养流程
保养分为两个步骤，玩家依次完成：

##### 步骤一：物理修复（磨损度）
- 画面：魔偶坐在维修台上，战损部位高亮显示
- 玩家选择修复材料（廉价机油 / 中级润滑剂 / 高级花香护理液）
- 播放修复动画（焊接火花 / 擦拭 / 涂抹）
- 魔偶在修复过程中有反馈：
  - 廉价机油："好冷……但没关系。"（磨损度-30%，好感度-2）
  - 中级润滑剂："嗯，好多了。"（磨损度-45%，好感度不变）
  - 高级花香护理液："好香……主上，谢谢你。"（磨损度-60%，好感度+5）

##### 步骤二：瘴气净化（侵蚀度）
- 画面：魔偶身上的黑色瘴气可视化显示
- 玩家选择净化材料（净化结晶 / 高纯净化结晶）
- 播放净化动画（玩家用刷子擦洗瘴气，瘴气逐渐化为蒸汽散去）
- 净化完成后魔偶立绘焕然一新，对比效果强烈

#### 保养的时间投入感
保养动画不应该是"按一下跳过"的瞬间操作。修复+净化的完整流程应该有**15~30秒的交互动画**，让玩家感受到"我在花时间照顾她"。

如果玩家选择跳过动画（允许快进），好感度增量减半。

> **设计意图**：保养是"用时间换情感"的交互。如果能一键完成，它就变成了纯粹的数值操作。保留动画时长，是让玩家在每一次保养中都重新面对"她是一个需要照顾的人"这个事实。

### 2.5 特殊交互（Special）

这些交互不在常规菜单中，而是由特定条件触发的一次性或稀有事件。

#### A. 擦拭污渍
- **触发条件**：从深渊回来后的第一次进入工坊界面（魔偶身上有战斗留下的泥渍/血渍）
- **操作**：玩家按住鼠标在魔偶身上滑动，擦去污渍。每擦净一处触发一句反馈
- **效果**：微量好感度+3~5，SAN恢复5。这是一个"无缝嵌入"的交互，不消耗AP
- **设计意图**：让玩家在深渊后的第一刻就产生"我回来了，她还在"的安心感

#### B. 噩梦安抚
- **触发条件**：SAN值低于20% + 夜晚 + 进入工坊时
- **表现**：画面变暗，魔偶蜷缩在角落，低声呜咽
- **操作**：点击她 → 触发"安抚对话"（2~3轮选项）→ 正确安抚后她平静下来
- **效果**：SAN恢复15~20，好感度+10。如果选项选错（如"别哭了，浪费时间"），好感度-5
- **出现频率：** 每周最多触发1次

#### C. 生日/纪念日事件
- **触发条件**：每个魔偶有一个隐藏的生日设定。该日期到来时自动触发
- **表现**：进入工坊时，魔偶穿着特殊服装（如果有的话），台词为专属庆祝内容
- **操作**：送礼时好感度获取×3（限当天）。不送礼无惩罚，但会有一句落寞的台词
- **设计意图**：给长期游玩的玩家一个惊喜。如果玩家碰巧记住了日期并提前准备了礼物，这种"被记住"的感觉是无价的

#### D. 告别前夜
- **触发条件**：游戏进入最终月最后一周
- **表现**：夜晚AP③时，魔偶主动发起对话。内容根据全局状态（好感度/结局走向）大幅不同
- **操作**：纯对话，无选项。她说的话是对这一整局游戏经历的总结
- **效果**：无数值变化。这是纯粹的叙事时刻
- **设计意图**：在最终结局前给玩家和魔偶一个"最后的夜晚"

## 三、交互的反馈体系

### 3.1 魔偶的情绪状态机
魔偶在工坊中的表现由一个**情绪状态机**驱动，当前情绪由好感度+SAN+近期事件综合决定：

| 情绪状态 | 触发条件 | 表现特征 |
|---|---|---|
| **元气** | SAN > 80% + 好感度Lv.4+ | 主动搭话、眼神追踪鼠标、偶尔哼歌 |
| **平静** | SAN 50~80% + 好感度Lv.2~4 | 安静待机、被点击时正常回应 |
| **疲惫** | SAN 30~50% | 低头、动作慢、被点击时反应迟钝 |
| **压抑** | SAN < 30% 或好感度Lv.1 | 蜷缩、避免眼神接触、触摸时退缩 |
| **崩溃边缘** | SAN < 10% | 颤抖、偶尔低声自语、触摸脸颊时可能尖叫 |
| **冰冷抗拒** | 好感度冰点 | 背对玩家、不回应任何触摸、对话只有单字 |

### 3.2 语音系统
- 魔偶的语音不是独立的系统，而是交互反馈的一部分
- 每句语音与特定交互+情绪状态绑定
- 同一种交互在不同情绪下触发不同语音（如触摸头顶：元气时是"嘿嘿"，压抑时是沉默）
- 语音总量控制在可控范围内，优先录制高频交互的多状态变体

### 3.3 立绘差分
- 魔偶的立绘不是一张静态图，而是根据情绪状态和交互行为切换的多层差分
- 差分维度：表情（6~8种）× 姿态（3~4种）× 状态叠加（战损/瘴气/换装）
- 在保养交互中，立绘实时从"战损态"变为"修复态"——这个视觉对比是保养满足感的核心来源

## 四、交互频率与防滥用设计

### 4.1 每日交互上限

| 交互类型 | 每日上限 | 原因 |
|---|---|---|
| 触摸 | 前5次有效，之后无好感变化 | 防止无脑点击刷好感 |
| 对话 | 受AP限制（通常1~2次） | 时间是最稀缺的资源 |
| 赠礼 | 无次数限制，但边际递减极快 | 第2件起效率×0.3，第3件起×0.1 |
| 保养 | 受AP + 材料限制（通常1次） | 自然约束 |
| 特殊交互 | 触发条件限制 | 不可刷取 |

### 4.2 防"功利化"设计
如果交互只是"好感度ATM机"，玩家会把它当作需要最优解的效率问题。以下设计确保交互保留情感温度：

- **触摸无成本但收益极低**——它的价值在于"存在"而非"产出"
- **赠礼边际递减**——砸钱暴刷不划算，不如细水长流
- **保养动画不可跳过（跳过则减半）**——强迫玩家花时间"看着她"
- **对话随机池**——无法通过SL大法刷到完美回答

## 五、交互与其他系统的联动

### 5.1 交互 → 特质系统 (GDD_09)
| 交互行为 | 可能触发的特质 |
|---|---|
| 长期高频触摸 + 高好感 | 「信赖之心」的触发条件之一 |
| 长期不做任何交互 | 「冰冷服从」的触发条件 |
| 保养时总选廉价材料 | 「心墙」的风险因子之一 |
| 赠礼全是贸易品（值钱但不走心） | 「工具人觉悟」的风险因子之一 |
| 连续赠送情感锚点物 + 夜晚陪伴 | 好感度加速突破的组合奖励 |

### 5.2 交互 → 房间系统 (GDD_11)
| 交互行为 | 房间中的体现 |
|---|---|
| 首次送花类礼物 | 窗台出现干花 |
| 首次送出旧武器 | 墙上挂出旧武器 |
| 好感度首次达Lv.5 | 床头出现合影 |
| 好感度达Lv.8 | 天花板出现星星贴纸 |

### 5.3 交互 → 深渊表现 (GDD_01/02)
交互的积累会在深渊中以意想不到的方式回馈：
- 好感度足够高时，安全区通讯的对话更加温暖，魔偶主动提供战术建议
- 好感度Lv.5以上解锁致死保护的"奇迹机制"（详见GDD_03）
- 累计保养次数影响深渊中魔偶受伤时的语音——从"无所谓"到"没关系，因为主上会修好我的"

## 六、边界规则

| 情况 | 规则 |
|---|---|
| 魔偶处于「崩溃事件」中（SAN=0强制停摆），能交互吗？ | 只能触摸（且反馈是应激的）。不能对话、赠礼、保养——必须先通过日程安排处理崩溃事件 |
| 保养到一半玩家退出游戏 | 保养进度不保存，材料不消耗。下次重新开始保养流程 |
| 玩家从开局到结局从未做过任何交互 | 完全允许。好感度将极低或冰点，魔偶会呈现「冰冷服从」状态。通往"无情兵器"结局线。这也是一种合法的游玩方式 |
| 玩家疯狂送诅咒物品给魔偶 | 每次扣好感度。连续赠送3次诅咒物品后触发强制对话："主上，你是故意的吗？"此后诅咒物品的赠礼入口被锁定1周 |
| 好感度已满（Lv.10），交互还有意义吗？ | 好感度不再增长，但对话池解锁最高等级的专属内容。触摸反馈变为最亲密版本。满好感的交互不是"没用"，是"享受成果" |
