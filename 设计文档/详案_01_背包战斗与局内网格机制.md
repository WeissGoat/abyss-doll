# 详案_01：背包战斗与局内网格机制设计规范 (GDD_01)

> **定位说明：** 本文档是战斗表现、背包规则和数据结构的落地标准。程序和美术可直接根据本文档的数据表现规范进行游戏核心原型的开发。

---

## 第一部分：UI 层与交互逻辑（表现层）

本设计屏弃了传统卡牌发牌思路，将所有信息密度集中在“物理网格交互”上。

### 1. 核心屏幕分区（布局设计）
屏幕以大约 4:6 的比例划分为两大区域：
- **【左侧区：战斗可视化舞台】**
  - **角色态：** 显示魔偶Live2D/Spine骨骼动画，下方显示两大核心条：`HP(血盾)` 与 `SAN(理智)`。
  - **敌人组：** 显示当前节点遭遇的敌人列表及意图（下一步准备造成的伤害或上的异常状态）。
  - **战利品暂存槽 (Inbox)：** 左下角为“未入包战利品区”。刚打完怪掉落的东西会悬停在这里。如果不装进包里直接继续走，会直接被销毁。
- **【右侧区：核心书写板（背包网格矩阵）】**
  - **画布大小：** 初始为 4x4 或 4x5 的方格型网格底盘。
  - **网格材质：** 正常格子（亮色）、被诅咒结晶锁死的格子（紫黑色且带有晶体特效，无法放置物品）。

### 2. 玩家输入控制与时间流派
战斗是**“半即时推进（类似 ATB）”**，但对玩家交互做了极高容错的宽容处理。
- **基础时间态：** 自动进行，武器根据自身的进度条（或齿轮特效）自动充能，充填满后自动开火。
- **子弹时间（拖拽降速）：**
  - 玩家**鼠标左键按住**背包内的任意道具，或准备将战利品槽的东西拿进包里时，触发【子弹时间】（`TimeScale = 0.05`）。
  - 为什么不完全暂停？因为需要保留一点战斗节奏紧迫感。在这极慢的时间里，玩家能够专注玩俄罗斯方块拼图。
- **放置反馈 (Placement Feedback)：**
  - 拖拽悬停时，合法的格子高亮显示为绿色；如果存在重叠或碰到“晶化异常死格”，高亮显示为红色并阻止放下。
  - 成功放下瞬间，如果该物品连结（Aura）触发了旁边的武器增益，播放“连通特效（如发光的电路连接线）”以及清脆的金属共鸣音效。

## 第二部分：核心流转机制与法则（业务逻辑层）

战斗底层的计算框架完全遵循 ECS（实体组件系统）或强解耦的信号模式。

### 1. 物品运行法则与充能池
网格内的每一个具有攻击能力的物品被视为一个**独立发射器 (Emitter)**。
- **进度更新 (`Tick`)：**
  ```python
  # 伪代码：每个 Tick 推进进度
  FillProgress = min(MaxFill, FillProgress + DeltaTime * BaseSpeed * AuraSpeedBuff)
  if FillProgress >= MaxFill:
      ExecuteFire(ItemData)  # 开火
      FillProgress = 0       # 归零
  ```
- **疲劳值/耐力池锁定：** 如果开火需要消耗魔偶自身能量（如笨重的近战武器耗耐力为 2），开火瞬间检测能量槽。若不够，武器进度条卡在 100% 变灰，进入 `State_WaitStamina`（等待耐力恢复）状态。

### 2. 形态与空间碰撞算法
- 需要一个核心类 `GridManager`，以二维数组（如 `int[10, 10]`）维护空间状态。
- `0` 为空，`1` 为被物品占据，`-1` 为被深渊结晶锁死（不可占用，不可交互）。
- **坐标系定位：** 所有物品以其自身形状矩阵的左上角作为 `(X, Y)` 原点。如果在翻转系统解锁时，需支持网格数组根据矩阵逆时针旋转90度的算子。

### 3. 光环生效扫描周期 (Aura Scan Pipeline)
为了避免无意义的全局刷新，引入严格的更新周期。
- 什么时候会触发刷新？ -> 触发 `Event_Inventory_Changed`（放入、拿起、销毁）。
- **扫描逻辑：** 
  1. 清空当前网格所有叠加的附属属性（Buffs 清零）。
  2. 遍历所有带发光标签（Aura/Connector）的物品。
  3. 检查它指明的物理方向（如：[上方1格] 或 [周围8格]）。
  4. 对该覆盖区域内的目标进行验证。如果目标存在并带有 `RequireTag` 要求的标签，则挂载乘区参数。
  *(这套设计杜绝了复杂循环推导导致死锁的风险。)*

## 第三部分：原型白盒样例库 (MVP Demos)

开发第一个可运行的Demo时，使用以下三件装备、一件辅助插件和一只怪物测试完整框架循环。

### 【展示用物品清单】

#### 物品 1：破旧的发条剃刀 (Melee Weapon)
- **形状 (GridShape)：** 1x2 (单列2格高度)
- **属性：** 基础充能 2.0 秒，造成物理伤害 3。每次挥击消耗玩家全局盾牌耐久 0.5。
- **标签 (Tags)：** `[Melee]`, `[Gear]`

#### 物品 2：电磁引爆器 (Magic Weapon)
- **形状 (GridShape)：** 2x2 (方块结构)
- **属性：** 基础充能 4.5 秒，造成群攻魔法伤害 8。
- **标签 (Tags)：** `[Magic]`, `[Heavy]`

#### 物品 3：锈蚀导能管 (Accessory - 模型A流派的核心)
- **形状 (GridShape)：** 1x3 (水管形状)
- **属性：** 本身无任何充能动作和输出。
- **光环效果 (Aura)：** 只要其左右相接触（哪怕只接触它长边的一格）的所有带 `[Melee]` 标签武器，每秒自动回填 0.5 秒的进度（超级加速）。
- *设计意图：* 逼迫玩家将其横置于武器中间，制造极限攻速平A流。

### 【展示用怪物库】

#### 怪物测试机：被污染的史莱姆包囊
- **HP：** 50，**护甲：** 无。
- **行为序列：** 
  - (1) 普通攻击：每 3 秒发起一次撞击（魔偶 HP-2）。
  - (2) 特效攻击 (核心干扰)：每 8 秒朝天上喷吐毒液。毒液会在 1 秒后随机砸中玩家背包里两个为空的“1x1”格子。
  - **恶心机制：** 这两个空位会被“剧毒凝胶物”填充。它们会挤占空间且无法立刻消除。如果玩家想要清理，只能用鼠标拖出来丢弃，拖出来时还会遭到“每丢弃一个扣 1 点 SAN 值”的痛点惩罚。

## 第四部分：异常与容错处理 (Edge Cases)

| 极端情况 (Edge Case) | 系统裁决处理规范 (Resolution Rule) |
| :--- | :--- |
| 拖动物品 A 悬停在半空（未放下）时，充能时间到了吗？ | **时间卡死：** 物品悬空状态时，其内部定时器冻结。只有放入包里才会重新走进度。 |
| 怪物刚把某格锁死（结晶化），但我手里的东西刚好准备往下放？ | **放置驳回：** 拖拽放下的瞬间（`OnMouseRelease`），重新检测底盘数组状态。若有重叠物或 `-1`锁死格，物品退回到鼠标位置，触发屏幕红色震动反馈。 |
| 魔偶耐力/体力为 0 且无法自然恢复，但武器满进度一直开不了火？ | **憋死反噬：** 连续超过 5 秒若武器处于满进度但由于能源不足无法开火，武器将“过载”。其进度条退回 0%，并导致与其相邻的十字内格子产生一次爆炸（伤害玩家自身 5HP）。用来惩罚贪刀却不想好能源后路的玩家。 |
| 刚从战利品区拿进来的武器，其进度条如何初见？ | **初始冷却：** 从外面带入并在战斗中当场拖进网格的武器（现场换装），充能进度强制从 0 开始，且挂载一次 `FreshDrop` (首轮充能冷却时长增加 50%) Debuff。这是为了防止玩家利用战利品槽作为“卡冷却”的副武器包无线换枪。 |
