# 详案_05：剧本调度引擎与世界观封装逻辑 (GDD_05)

> **定位说明：** 不使用传统死板的“第几关播第几段动画”系统。本系统完全基于**“环境与事件钩子”**驱动，把极具魅力的世界观碎片化地撒在背包、地图和账单的每个角落。

---

## 第一部分：UI 层与叙事表现形式（表现层）

废除了大量站桩对话，将剧情表演融合进系统交互的三种形式。

### 1. AVG 沉浸式弹窗 (用于关键主线)
- **屏幕表现：** 屏幕变暗，在屏幕下方弹出经典的半透明黑底对话框。两侧显示角色大半身立绘，带呼吸效果。
- **打字机与跳过：** 文字逐字显示（可调速），并且伴有魔偶特有的“电子杂音/系统提示音”音效。
- **适用场景：** 魔偶好感度突破、交租日老板逼债、达成游戏隐藏结局。

### 2. 状态机浮动轻语 (Bubble Text)
- **屏幕表现：** 不需要玩家点击确认。在小镇等待界面、或是深渊地图选节点时，魔偶头上或是屏幕右上角直接冒出的气泡文字，停留 3 秒后淡出。
- **适用场景：** 纯粹的环境叙事。例如，当她理智值极低时，会在走图时自言自语：“主上……我的左眼好痛……有一双眼睛在看我”。极大地增强代入感而不打断心流。

### 3. 物品考古图鉴 (Lore Info)
- **屏幕表现：** 鼠标悬停在物品上按 `[Alt]` 键展开详细。除了原本的战斗网格数值外，下方附带长段的档案信息。
- **适用场景：** 深渊废土世界观的补充。比如“锈蚀的怀表”不仅是一个 1x1 的素材，下面还写着“悬丝镇第12突击队队长的遗物，上面刻着他女儿的名字”。

## 第二部分：事件总线引擎 (Event Bus Engine)

这套系统的核心在于：**“任何事都在监听，满足条件直接乱入”**。

### 1. 全局进度旗帜 (Global Flags)
游戏后台维护一个长长的字典。
例如：`{"has_met_boss": true, "days_survived": 12, "doll_death_count": 2}`。这是判断玩家到底“走到哪一步”的唯一依据。

### 2. 触发器钩子配置规则 (Trigger Rules)
剧情不是按时间线写的，是按“条件”写的。
- **时间钩子 `OnDayEnd`：** 每过一天，系统进行自检。如果 `Day == 30` 且 `flag_rent_paid == false`，马上塞入一段“债主强行踢门讨债”的对话树。
- **空间钩子 `OnNodeEnter`：** 当玩家在深渊进入深度为 20（最底端）的节点时，如果 `flag_seen_truth == false`，触发最重磅的主线揭秘对话。
- **状态钩子 `OnSanityChanged`：** 这是本作最精髓的设计。监听 SAN 值。当 SAN < 10 持续时长超过 3 天，触发一段隐藏好感剧情“魔偶半夜敲响你的房门乞求安慰”。

### 3. 链表式对话执行 (Dialog Nodes)
如果满足了触发器，就开始执行具体的剧本文件。
采用单链表结构，每个句子是一个节点，包含【下一句话的 ID】或【几个让你选的分支选项】。

## 第三部分：原型样例库 (MVP Demos)

开发原型时直接硬编码以下两段经典的“黑深残”剧情用例。

### 1. 强制经济挤压事件（环境压迫）
- **触发条件：** `Day == 15`，且 `PlayerGold < 500` (钱不够交第一次租金)。
- **发生时点：** 玩家当晚在工坊点击“结束今天”时强制打断。
- **剧本树：**
  - **[债主]**：“听说你最近在深渊发财了？怎么连这点保护费都凑不齐？”
  - **[玩家选项 1]**：【破财消灾(上交500金币)】 -> 系统直接报错，因为你钱不够（置灰）。
  - **[玩家选项 2]**：【恳求宽限】 -> 债主冷笑，触发指令：强制没收玩家仓库里价值最高的三个（带有 `Valuable` 标签的）战利品充公。
- **设计目的：** 用极不讲理的抢劫，把玩家逼向绝境，建立对底层资本的仇恨。

### 2. 战利品带来的碎玻璃刀子（情感牵挂）
- **触发条件：** 玩家成功将特定物品 `[陈旧八音盒(占2x2格，剧毒属性)]` 存活带出深渊，并在工坊将其作为礼物送给好感等级 >=3 的魔偶。
- **发生时点：** 拖送成功的瞬间。
- **剧本树：**
  - **[魔偶]**：“主上，这首歌……为什么我在听到的时候，会有液体从视觉传感器里流出来？”
  - **[系统指令]**：播放悠扬悲伤的 BGM，画面变为前世地球毁灭前的闪回。
  - **[玩家选项 1]**：【“这是人类叫做‘眼泪’的东西”】
  - **[结果指令]**：魔偶所有负面特质清空，理智上限永久 +20。
- **设计目的：** 让玩家觉得在深渊里“少放一把枪，死也要把这个占地方又没用的破烂八音盒带回来”的艰难决定，是彻底值得的。

## 第四部分：异常与边界情况 (Edge Cases)

| 极端情况 (Edge Case) | 系统裁决处理规范 (Resolution Rule) |
| :--- | :--- |
| **同一时刻触发了“交租日剧情”和“好感度突破剧情”，哪个先播？** | **事件队列与优先级 (Event Queue)：** 所有事件触发器自带 `Priority`（优先级）权重参数。系统会将高权重的生存危机（交租/死亡提醒）放在队列最前面播放。播完后，好感度事件会被“静默存留”，等玩家下一次回到默认待机状态时再弹出。 |
| **玩家疯狂点击空格/鼠标左键，全速跳过了所有剧情演出** | **指令强执行 (Silent Execution)：** 对话系统不可随意被摧毁进程。即使文字没有播完，由于跳过导致该节点直接销毁时，对话附带的系统奖励/惩罚指令（如 `Reward_Gold 500`）也必定会被执行验证类收容，不会因为跳过而错失核心物资。 |
