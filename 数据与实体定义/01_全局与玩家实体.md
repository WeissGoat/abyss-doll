# 全局与玩家实体定义 (Player & Global Entities)

> 本文档定义了游戏最顶层的数据结构，掌管玩家的总资产、游戏时间和宏观进度。

## 1. 玩家档案 (PlayerProfile)
玩家账号层级的实体，独立于具体的人偶，掌管不可丢失的资产与进度。

```json
{
  "entity": "PlayerProfile",
  "description": "玩家全局档案数据",
  "properties": {
    "UID": "string // 玩家唯一标识",
    "Money": "int // 当前持有的金币数量",
    "DebtLevel": "int // 当前所处的债务阶段(1=初级, 2=中级...)",
    "WorkshopLevel": "int // 工坊等级，决定可解锁的功能与蓝图上限",
    "MemoryFragments": "int // 记忆碎片，唯一不可被抢夺的局外天赋资源",
    "ActiveDollID": "string // 当前上阵（激活）的人偶ID",
    "UnlockedDolls": "List<string> // 已解锁的人偶ID列表",
    "StashInventory": "List<ItemEntity> // 地面大仓库（非网格，纯列表存储）"
  }
}
```

## 2. 时间与日程引擎 (TimeState)
控制游戏宏观推进的计时器，也是资源博弈的核心轴。

```json
{
  "entity": "TimeState",
  "description": "时间与日程状态",
  "properties": {
    "TotalDaysSurvived": "int // 总计存活天数（核心死亡线指标）",
    "CurrentMonth": "int // 当前月份 (1-12)",
    "CurrentDay": "int // 当前日 (1-28)",
    "CurrentAP": "int // 当日剩余行动点数（每日刷新）",
    "CurrentPhase": "DayPhase // 当前所处的时间段"
  }
}
```

### 2.1 相关枚举与可选项 (Enums)

```cpp
enum DayPhase {
    Morning_Prep = 0, // 备战期：安全屋整理、采买、摸头（零时耗）
    Afternoon_Action = 1, // 核心抉择期：下深渊或停工疗愈（消耗AP与天数）
    Evening_Settle = 2 // 结算期：营业卖货、强制扣租金、触发过天事件
}
```
