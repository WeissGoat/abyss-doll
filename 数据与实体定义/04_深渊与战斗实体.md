# 深渊与战斗实体定义 (Dungeon & Combat Entities)

> 本文档定义了“打与撤”环节的数据结构。地图的生成、节点的抉择以及怪物的数值都在这里。

## 1. 深渊单局推演实体 (DungeonRunEntity)
每次点击“下深渊”产生的一个临时生命周期对象。

```json
{
  "entity": "DungeonRunEntity",
  "description": "单次深渊探索记录",
  "properties": {
    "RunID": "string // 探索实例ID",
    "CurrentLayer": "int // 当前所处深度层数",
    "TotalSANCalled": "int // 累计消耗/损失的理智值",
    "NodesVisited": "int // 走过的节点数量",
    "CurrentMapTree": "DungeonNode // 当前树状图的根节点指针",
    "TempInventory": "List<ItemEntity> // 局内安全盒（即使战败也能带出的极小容量背包）"
  }
}
```

## 2. 地图节点实体 (DungeonNode)
构成路线选择的基本单位。

```json
{
  "entity": "DungeonNode",
  "description": "深渊地图节点",
  "properties": {
    "NodeID": "string // 节点唯一ID",
    "NodeType": "NodeType // 节点类型",
    "SANCost": "int // 经过此节点需要支付的'过路税'(理智值)",
    "IsRevealed": "bool // 是否在迷雾外可见",
    "NextNodes": "List<DungeonNode> // 分支路线（子节点列表）",
    "EventConfigID": "string // 若是事件/战斗节点，对应的配置表ID"
  }
}
```

## 3. 战斗怪物实体 (MonsterEntity)
战场上的敌人。由于战斗也是基于背包体系，怪物的数值也需要具备空间破坏力。

```json
{
  "entity": "MonsterEntity",
  "description": "敌对怪物对象",
  "properties": {
    "MonsterID": "string // 怪物配置ID",
    "HP": "int // 怪物血量",
    "AttackCooldown": "float // 攻击间隔",
    "DamageValue": "int // 基础伤害值",
    "GridInterference": "GridAttackType // 网格干涉能力（比如：锁定玩家背包某几个格子）"
  }
}
```

## 4. 相关枚举与可选项 (Enums)

```cpp
enum NodeType {
    Entrance = 0,   // 入口
    Combat = 1,     // 普通战斗 (产出普通素材/战利品)
    Elite = 2,      // 精英战斗 (高压，高产出)
    Event = 3,      // 随机事件 (SAN值检测、扔骰子)
    SafeRoom = 4,   // 安全屋 (提供少量的疗愈，并允许安全撤退)
    Boss = 5        // 关底Boss
}

enum GridAttackType {
    None = 0,           // 纯打数值血量
    LockCell = 1,       // 吐丝/结冰：暂时锁死玩家背包X个格子，使其内物品技能停摆
    AddCursedItem = 2,  // 塞垃圾：强行往玩家空位塞入不可丢弃的毒属性战利品
    Shuffle = 3         // 打乱：强行改变玩家背包内部分物品的位置或旋转方向
}
```
