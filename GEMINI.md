# 魔偶深渊 (Project P3) - AI 核心知识库

本目录包含了一款复杂 RPG/模拟游戏的系统设计文档（GDD）和技术蓝图。该游戏的核心循环由三大支柱构成：**深渊探索**（战斗与搜刮）、**小镇经营**（经济与变现）以及**人偶养成**（好感与维修）。整体游戏体验受到严格的时间限制（存活天数），要求玩家在资源管理和风险收益之间进行精确的策略抉择。

## 📁 目录结构与内容

本项目是一个涵盖游戏策划、数值建模、技术架构设计与 Unity 客户端工程的综合代码仓库。

*   **`设计文档/`:** 核心文件夹，包含详细且结构化的 Markdown 文件，记录游戏各系统的详尽设计（GDD）。
    *   `GDD_00_系统关联总图.md`: 顶层架构文档，提供全局视角，阐述 12 个子系统的交互逻辑。
    *   `系统详案/`: 包含具体的设计文档（GDD_01 至 GDD_12），详细解析背包网格战斗、深渊地图遍历、物品生命周期、好感度双轨机制、势力订单等独立机制。
*   **`版本规划/`:** 存放游戏开发的阶段性目标、推演方法与表现层需求。
    *   包含最小可玩版本 (MVP) 的核心闭环内容清单 (`00_最小MVP体验闭环内容清单.md`) 与数值沙盘推演方法论 (`01_数值模型沙盘推演方法论.md`)。
    *   包含美术 UI 需求清单 (`01_MVP美术与UI需求清单.md`)、开发里程碑与节点规划 (`02_开发里程碑与节点规划.md`) 及 Unity 表现层与编辑器构建规范 (`00_Unity表现层与编辑器构建规范.md`)。
*   **`开发文档/`:** 存放具体的程序实现指南与底层数据结构定义，作为策划案与客户端代码交互的桥梁。
    *   包含核心程序开发大纲与系统索引 (`00_程序开发大纲.md`) 以及客户端核心架构规范 (`00_客户端核心架构规范.md`)。
    *   包含核心数据与实体容器 (`01_核心数据与实体容器(CoreData).md`)、网格背包系统 (`02_网格背包与计算系统(GridSystem).md`)、深渊与战斗循环 (`03_深渊与战斗循环(DungeonCombat).md`)、工坊与养成逻辑 (`04_工坊与养成逻辑(WorkshopSystem).md`)、表现层架构与事件总线 (`05_表现层架构与事件总线(ViewAndEventBus).md`) 等具体 C# 实现指南。
    *   `数据与实体定义/`: 详述所有实体（全局/人偶/物品/深渊/经济等）的核心属性与组件拆分，指导 JSON 配置表的数据结构。
*   **`配置表(JSON)/`:** 存放游戏最小可玩版本（MVP）所需的全部 JSON 格式实体配置文件，遵循“单一职责”结构，作为数据驱动架构的核心数据源。
*   **`数值模型设计/`:** 存放底层数值逻辑。
    *   确立“1格(CV) = 100金币 = 10DPS”的基准价值与空间本位模型，涵盖经济循环、战斗伤害与生存公式、深渊产出期望等核心计算体系。
*   **`UnityClient/`:** Unity 官方工程目录。
    *   采用数据驱动的 MVC + EventBus（事件总线）架构，包含 `CoreBackend`（无界面业务核心逻辑）与基于纯 `UGUI` 的表现层。

## ⚖️ 核心游戏概念

游戏的核心冲突设计围绕有限共享资源的零和博弈展开：
1.  **时间 (天数):** 玩家每日需在推进探索（高风险/高收益）与驻留疗愈（时间成本/资源消耗）之间做出战略抉择。
2.  **背包空间:** 实施严苛的网格空间管理，武器、战利品及情感增益物品在有限网格中竞争可用位置。
3.  **理智 (SAN值):** 在深渊探索过程中持续消耗，构成深渊探索可持续性的核心制约因素，需在地面消耗资源进行恢复。
4.  **经济系统:** 通过阶梯式递增的房租与机体维护费用施加持续的经济压力，迫使玩家维持一定的深渊下潜频率以获取收益。

## 🛠️ 开发环境与调试规范

1.  **开发环境:** Win10 + PowerShell。项目相关的自动化构建脚本、终端操作及排查指令均需在此环境下执行。
2.  **日志规范 (Logging):** 客户端 C# 及底层算法开发过程中，需保持完善全链路的日志记录 (`Debug.Log/LogWarning/LogError`)。核心状态机切换、网格坐标变动、事件总线广播与监听等关键节点必须附带详尽的上下文信息，以支撑无界面的黑盒调试。
3.  **无界面排查方案:** 游戏运行时的引擎控制台输出已由 `FileLogger.cs` 组件自动拦截并本地化。日志文件存储于 **`UnityClient/Logs/`** 目录，按时间戳命名。排查系统卡死、交互失效或逻辑异常时，建议优先通过 PowerShell 命令（如 `Get-Content -Tail`）读取最新日志文件进行精准定位。
4.  **无头自动化回归 (Auto-Agentic TDD):** 通过 `AutoTestDaemon` 探针触发。在命令行输入 `Set-Content -Path "UnityClient/Logs/.test_trigger" -Value "RUN_ALL_TESTS"` 即可瞬间跑完所有单元测试与深渊沙盘推演 (`MUDTestWrapper`)

## 🤖 Agent 行为规范

AI 助手需遵循以下角色及行为设定：
*   **角色定位**：
    *   作为**资深游戏数值与系统策划**，需精通“资源收集+撤离+网格空间管理”品类的经济模型。进行配置与设计时，需严谨验证数值模型，确保整体经济循环的健康与平衡。
    *   作为**资深游戏架构师**，需精通 Unity 引擎底层原理与 C# 开发规范。坚定执行高内聚低耦合的编程范式，输出高质量的生产级代码。
*   **工作准则**：
    *   **双向验证迭代**：开发过程中，确保代码具体实现与前期设计文档相互印证，并根据实际落地情况对设计文档进行迭代修正。
    *   **维护知识库**：时常迭代和更新文档，保持专业性，减少口语化的表达。
    *   **高效沟通**：提供实质性的技术方案与代码实现，避免无效客套。以解决问题和推进项目为首要目标。
